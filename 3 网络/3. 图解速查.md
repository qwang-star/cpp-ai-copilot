## 一、三次握手（全细节）

```
  客户端 (CLOSED)                    服务端 (CLOSED)
    │             SYN_SENT                 │ LISTEN
    │                                       │
    │ ① SYN=1, Seq=x                        │
    │    (无ACK标志, Ack无意义)               │
    │ ──────────────────────────────────→   │
    │                                       │ SYN_RCVD
    │                                       │
    │ ② SYN=1, ACK=1, Seq=y, Ack=x+1       │
    │    (同时SYN+ACK)                       │
    │ ◁──────────────────────────────────   │
    │                                       │
    │ ③ ACK=1, Seq=x+1, Ack=y+1            │
    │    (纯ACK, 可携带数据)                 │
    │ ──────────────────────────────────→   │
    │                                       │ ESTABLISHED
    │ ESTABLISHED                           │

    状态变化:
    C: CLOSED → SYN_SENT → ESTABLISHED
    S: CLOSED → LISTEN → SYN_RCVD → ESTABLISHED
```

## 二、四次挥手（全细节）

```
  客户端                              服务端
    │ ESTABLISHED                         │ ESTABLISHED
    │                                     │
    │ ① FIN=1, Seq=u, Ack=v               │
    │    (关闭客户端→服务端方向)             │
    │ ──────────────────────────────────→ │ CLOSE_WAIT
    │ FIN_WAIT_1                           │
    │                                     │
    │ ② ACK=1, Seq=v, Ack=u+1             │
    │    (确认收到FIN, 但数据可能没发完)     │
    │ ◁────────────────────────────────── │
    │ FIN_WAIT_2                           │
    │                                     │ (发完剩余数据)
    │                                     │
    │ ③ FIN=1, ACK=1, Seq=w, Ack=u+1      │
    │    (服务端也发完了)                    │ LAST_ACK
    │ ◁────────────────────────────────── │
    │                                     │
    │ ④ ACK=1, Seq=u+1, Ack=w+1           │
    │    (最后确认)                         │
    │ ──────────────────────────────────→ │ CLOSED
    │ TIME_WAIT (等2MSL)                    │

    状态变化:
    C: ESTABLISHED → FIN_WAIT_1 → FIN_WAIT_2 → TIME_WAIT → CLOSED
    S: ESTABLISHED → CLOSE_WAIT → LAST_ACK → CLOSED
```

## 六、HTTPS TLS 握手（TLS 1.2）

```
  客户端                              服务端
    │                                    │
    │ ① ClientHello                      │
    │   支持的加密套件列表                  │
    │   TLS版本 + 随机数ClientRandom       │
    │ ──────────────────────────────────→ │
    │                                    │
    │ ② ServerHello                      │
    │   选定加密套件                       │
    │   随机数ServerRandom                 │
    │   证书(含公钥)                       │
    │   ServerHelloDone                   │
    │ ◁────────────────────────────────── │
    │                                    │
    │ ③ 验证证书合法性                     │
    │   生成 PreMasterSecret              │
    │   用公钥加密 PreMasterSecret         │

    │ ──────────────────────────────────→ │
    │                                    │
    │ ④ 用私钥解密得到 PreMasterSecret     │
    │   ClientRandom+ServerRandom         │
```

## 七、HTTP 版本演进对照

```
HTTP/1.0  每个请求一个TCP连接
          ┌──┐  ┌──┐  ┌──┐
          │R1│  │R2│  │R3│   3个请求 3个连接
          └──┘  └──┘  └──┘
          问题: 慢 (每次都要握手+慢启动)

HTTP/1.1  持久连接+管道化
          ┌──┬──┬──┐
          │R1│R2│R3│   共用1个连接，但响应必须有序
          └──┴──┴──┘
          问题: 队头阻塞 (R1慢就堵住R2、R3)

HTTP/2    多路复用
          ┌──┐
      ┌──→│R1│──→   1个连接，多个流并发
      │   └──┘      响应可以乱序返回
      │   ┌──┐
      ├──→│R2│──→   头部压缩 (HPACK)
      │   └──┘
      │   ┌──┐
      └──→│R3│──→   服务端推送
          └──┘
          问题: TCP层面的队头阻塞 (一个包丢,全部流卡住)

HTTP/3    QUIC over UDP
          各流独立,互不阻塞
          0-RTT握手 (快!)
          连接迁移 (切换网络不掉线)
```

## 八、URL 输入后的完整流程

```
用户输入: https://www.example.com/path

│
├─ 1. DNS 解析
│    浏览器缓存 → OS hosts → 本地DNS → 根→顶级→权威
│    example.com → 93.184.216.34
│
├─ 2. TCP 三次握手
│    SYN → SYN+ACK → ACK
│    客户端:随机端口 → 服务端:443
│
├─ 3. TLS 握手 (HTTPS)
│    ClientHello → ServerHello+证书 → 交换密钥
│    建立加密通道
│
├─ 4. 发送 HTTP 请求
│    GET /path HTTP/1.1
│    Host: www.example.com
│
├─ 5. 服务端处理
│    Nginx → 应用服务 → 数据库 → 返回响应
│
├─ 6. 接收 HTTP 响应
│    HTTP/1.1 200 OK
│    Content-Type: text/html
│    <html>...</html>
│
├─ 7. 浏览器渲染
│    解析HTML → 构建DOM → 加载CSS/JS → 渲染页面
│
└─ 8. TCP 四次挥手 (可能Keep-Alive复用连接)
     FIN → ACK → FIN → ACK
```

## 十、一句话速记卡片

| 概念 | 一句话 |
|------|--------|
| 三次握手 | SYN → SYN+ACK → ACK，防止旧连接 + 全双工确认 |
| 四次挥手 | FIN → ACK → FIN → ACK，全双工关闭 |
| TIME_WAIT | 等 2MSL，确保最后的 ACK 到达 + 旧报文消失 |
| 拥塞控制 | 慢启动指数→拥塞避免线性→3ACK砍半→超时回1 |
| HTTP/2 | 多路复用 + 头部压缩 + 服务端推送 + 二进制帧 |
| HTTPS | 非对称加密换对称密钥 |
| DNS | 域名→IP，默认 UDP 53 |
| epoll | 只拷贝一次 + 只返就绪 + 支持 ET |
| TCP粘包 | 字节流无边界 → 定长/分隔符/头+体 |
| Cookie/Session/Token | Cookie存浏览器/Session存服务端/Token自包含 |
