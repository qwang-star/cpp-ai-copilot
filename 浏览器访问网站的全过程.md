## 一、协议栈视角（理论模型）

这是计网课最常见的五层模型：

```
应用层
  │
  ├── HTTP
  │
传输层
  │
  ├── TCP
  │
网络层
  │
  ├── IP
  │
数据链路层
  │
  ├── Ethernet
  │
物理层
```

**它回答的是：数据被封装成什么格式？**

### 封装过程

浏览器发送：

```http
GET /hello HTTP/1.1
Host: localhost
```

会经历：

```
HTTP
  ↓
TCP
  ↓
IP
  ↓
Ethernet
  ↓
网卡发送
```

**每一层都加自己的头。**

最终在网线上跑的是：

```
┌─────────────┐
│ Ethernet 头 │
├─────────────┤
│ IP 头       │
├─────────────┤
│ TCP 头      │
├─────────────┤
│ HTTP 数据   │
└─────────────┘
```

这叫：**封装（Encapsulation）**。

**发送方向：上层 → 下层**

## 二、Linux 内核处理视角（服务器接收）

你看到的：

```
网卡
  ↓
IP 层
  ↓
TCP 层
  ↓
socket 缓冲区
  ↓
read()
```

**回答的是：服务器收到数据以后，内核怎么处理？**

这是一个**解包过程**。

### 完整链路

客户端发来：

```http
GET /hello HTTP/1.1
Host: localhost
```

到达服务器时，仍然是完整的帧：

```
┌─────────────┐
│ Ethernet 头 │
├─────────────┤
│ IP 头       │
├─────────────┤
│ TCP 头      │
├─────────────┤
│ HTTP 数据   │
└─────────────┘
```

网卡收到 Ethernet Frame，交给内核：

```
网卡
  ↓
Ethernet 层 → 剥掉 Ethernet 头
  ↓
IP 层       → 剥掉 IP 头
  ↓
TCP 层      → 剥掉 TCP 头
  ↓
socket 接收缓冲区
  ↓
read()
```

然后你的代码：

```cpp
read(fd, buf, 4096);
```

拿到的是：

```http
GET /hello HTTP/1.1
Host: localhost
```

**TCP 头已经被内核处理掉了，应用程序根本看不到。**

**接收方向：下层 → 上层**

## 三、完整过程：客户端 + 服务器

### 客户端（浏览器发送）

```
浏览器构造 HTTP 请求
  ↓
HTTP → TCP 封装
  ↓
TCP → IP 封装
  ↓
IP → Ethernet 封装
  ↓
网卡发出
```

这是 **上层 → 下层**。

### 网络中传输

网络里跑的是 **Ethernet Frame**，里面套着：

```
Ethernet Frame
  └── IP 包
       └── TCP 包
            └── HTTP 数据
```

### 服务器接收

```
网卡收到 Ethernet Frame
  ↓
Ethernet 层解析
  ↓
IP 层解析
  ↓
TCP 层解析
  ↓
放入 socket 接收缓冲区
  ↓
epoll 发现可读
  ↓
read()
  ↓
HTTP Parser
  ↓
HttpRequest 对象
```

这是 **下层 → 上层**。

最终得到：

```cpp
HttpRequest req;

req.method = "GET";
req.path   = "/hello";
req.headers["Host"] = "localhost:8080";
```

## 五、在 Linux Server 中的对应关系

```
浏览器
   │
   │ SYN
   ▼
内核 TCP 协议栈
   │
   │ SYN+ACK
   ▼
连接建立
   │
   ▼
accept()
   │
   ▼
read()
   │
   ▼
拿到 HTTP 文本
```

然后你的代码：

```cpp
HttpRequest req;
parse_http(buffer, req);
```

解析的是：

```http
GET /hello HTTP/1.1
Host: localhost:8080
```

而不是：

```
SYN
ACK
FIN
```

因为 **SYN/ACK 已经被内核 TCP 协议栈处理完了**，应用程序根本看不到。

## 六、面试要点

### Q: `read()` 读到的数据包含 TCP Header 吗？

**答案：不包含。**

```
网卡
  ↓
IP 层
  ↓
TCP 层       ← 内核在这里处理掉了 TCP 头
  ↓
socket 接收缓冲区
  ↓
read()       ← 你在这里读到的已经是纯应用层数据
```

当你：

```cpp
read(fd, buf, 4096);
```

时，得到的是：

```http
GET /hello HTTP/1.1
Host: localhost:8080
```

而不是 `TCP Header` + `HTTP Header` + `HTTP Body`。

TCP Header 已经被内核吃掉了。

### Q: `std::map<std::string, std::string> headers` 保存的是什么？

保存的是 **HTTP Header（应用层）**：

```
Host
User-Agent
Content-Type
Cookie
Authorization
Content-Length
...
```

而不是 **TCP Header（传输层）**：

```
SYN / ACK / FIN
seq / ack
window
...
```

## 七、核心总结

- **计网** = 数据怎么封装和传输
- **操作系统** = 数据到机器后怎么处理

而"浏览器访问服务器"这道经典面试题，本质上就是把这两条线合并成一条完整链路：

```
发送方向：HTTP → TCP → IP → Ethernet（上 → 下，封装）
接收方向：Ethernet → IP → TCP → socket → read() → HTTP Parser（下 → 上，解包）
```

它们是**同一件事的两个方向**。
