```
        ├──────────────────┤
```

## 三、epoll 内部结构

```
        用户态                    内核态
    ┌─────────────┐
    │ epoll_create │──→ ┌──────────────────┐
    │  返回 epfd   │    │   eventpoll 对象  │
    └──────┬──────┘    │  ┌──────────────┐ │
           │           │  │  红黑树(rbr)  │ │ ← 存储所有注册的 fd
    ┌──────▼──────┐    │  │  fd1→fd2→fd3  │ │   (epoll_ctl 增删改)
    │ epoll_ctl   │──→ │  └──────┬───────┘ │
    │ ADD/MOD/DEL │    │         │ 回调    │
    └──────┬──────┘    │  ┌──────▼───────┐ │
           │           │  │ 就绪链表(rdll)│ │ ← fd 就绪时内核把节点
    ┌──────▼──────┐    │  │  fd3→fd7     │ │   移到此链表
    │ epoll_wait  │◄──→│  └──────────────┘ │
    │ 返回就绪fd   │    └──────────────────┘
    └─────────────┘
    
    关键：只返回就绪的 fd，不遍历全部！O(1) 获取。
```

## 四、select vs epoll 文件描述符传递对比

```
select() 每次调用:                    epoll 模式:
                                     
  用户态          内核态              用户态           内核态
  ┌───┐                          ┌───┐
  │fd1│──拷贝──→  ┌───┐           │fd1│──epoll_ctl─→ ┌───┐
  │fd2│          │fd1│           │fd2│  (只一次)    │fd1│ (常驻)
  │fd3│          │fd2│           │fd3│              │fd2│
  └───┘          │fd3│           └───┘              │fd3│
                 └───┘                             └───┘
       ▲                                              │
       │ 每次调用都拷贝全部 fd!                          │
       │ 遍历全部分辨就绪: O(n)                         ▼
                                                   ┌──────┐
  epoll_wait:                                      │就绪链表│
  只返回有事件的 fd                                  │ fd2→ │
  不需要遍历全部: O(1)                               └──────┘
```

## 五、零拷贝：传统 vs sendfile

```
传统 read() + write(): 4 次拷贝，2 次 CPU 参与

  硬盘 ──DMA①──→ 内核缓冲区 ──CPU②──→ 用户缓冲区
                                        │
                                        │ CPU③
                                        ▼
  网卡 ◀──DMA④──  Socket缓冲区 ◀─────────┘


sendfile(): 2 次拷贝，0 次 CPU 参与

  硬盘 ──DMA①──→ 内核缓冲区 ──→ 只传描述信息(fd+offset+len)
                                        │
                                        ▼
  网卡 ◀──DMA②──  Socket缓冲区 ◀── gather DMA
                  
  注意：Socket缓冲区只存描述信息（指针+长度），
       实际数据直接从内核缓冲区 DMA 到网卡！
```

## 七、死锁四条件 → 破解对应

```
   条件① 互斥             条件② 持有并等待
   ┌─────────┐          ┌──────────────┐
   │ 资源独占  │          │ 拿着A等B      │
   │ 不能共享  │          │ 不放手        │
   └────┬────┘          └──────┬───────┘
        │ ← 这个无法破坏          │ ← 一次性分配
        │                       │    所有资源
        ▼                       ▼
    ┌───────────────────────────────────┐
    │          💀 死锁                  │
    └───────────────────────────────────┘
        ▲                       ▲
        │                       │
   ┌────┴────┐          ┌──────┴───────┐
   │ 不能被   │          │ A等B, B等C,  │
   │ 强制抢走  │          │ C等A → 环路  │
   └─────────┘          └──────────────┘
```

## 八、LT vs ET 触发行为

```
数据到达时间线:
  t1     t2     t3     t4
  ↓      ↓      ↓      ↓
 [500B] [500B] [500B] [500B]

LT 模式 (水平触发):
  epoll_wait → "有数据!" (500B) → 读了 300B → 还剩 200B
  epoll_wait → "还有数据!" (200B) → 读完
  epoll_wait → "有数据!" (500B) → ...
  每次 epoll_wait 只要缓冲区有数据就通知
  优点: 不会漏数据    缺点: 可能重复通知

ET 模式 (边缘触发):
  epoll_wait → "有数据!" (第一次就绪)
  必须循环读完直到 EAGAIN:
  while (read(fd, buf, n) > 0) { ... }
  下次数据到达才再次通知
  优点: 高效，只通知一次  缺点: 编程复杂，不用非阻塞会死锁!
```

## 九、CPU 缓存层次

```
        延迟           大小
    ┌─────────┐
    │   CPU   │
    ├─────────┤
    │ 寄存器   │  ~0 cycles    ~100B
    ├─────────┤
    │  L1     │  ~4 cycles    ~64KB
    ├─────────┤
    │  L2     │  ~12 cycles   ~256KB
    ├─────────┤
    │  L3     │  ~40 cycles   ~8MB (共享)
    ├─────────┤
    │  内存   │  ~100ns       ~GB
    ├─────────┤
    │  SSD    │  ~100μs       ~TB
    ├─────────┤
    │  磁盘   │  ~10ms        ~TB
    └─────────┘
    
    迟到代价:
    L1 miss → 多等 10ns (十几条指令)
    TLB miss → 多等 50ns (走页表)
    缺页     → 多等 10ms (读磁盘!) → 上千万条指令!
```

## 十、一句话速记卡片

| 概念 | 一句话 |
|------|--------|
| 进程 vs 线程 | 进程是资源单位，线程是调度单位 |
| 虚拟内存 | 每个进程看到独立的假地址，MMU 翻译成真地址 |
| TLB | 页表的 CPU 缓存，命中就不用查页表 |
| 缺页中断 | 访问不在内存的页 → 从磁盘加载 → 可能置换 |
| epoll | fd 只拷一次 + 只返回就绪的 + 支持边缘触发 |
| 零拷贝 | sendfile → 数据不经过用户态，2 次 DMA 搞定 |
| 死锁 | 互斥 + 持有等待 + 不可抢占 + 环路 → 四缺一不死锁 |
| 自旋锁 | 等不到就转圈（忙等），适合极短临界区 |
| 互斥锁 | 等不到就睡觉（阻塞），临界区长时用它 |
| 共享内存 | IPC 中最快的方式，因为不用内核拷贝 |
| brk vs mmap | malloc < 128KB 用 brk 调堆顶，>128KB 用 mmap 映射 |
