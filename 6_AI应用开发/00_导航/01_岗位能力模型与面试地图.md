# 01 岗位能力模型与面试地图

## 1. 岗位画像

后端 AI 应用开发的日常工作通常包括：

- 设计 AI 应用后端架构。
- 封装模型调用 SDK 或模型网关。
- 构建 RAG 知识库问答系统。
- 处理文档解析、切分、向量化、检索。
- 设计聊天、会话、任务、文件、知识库相关 API。
- 接入 Tool Calling，让模型能查询业务数据或执行动作。
- 做稳定性、限流、降级、日志、监控、评测。
- 处理 Prompt 注入、数据越权、敏感信息泄露等安全问题。

## 2. 能力模型

```text
编程实现能力
  -> 能写接口、查数据库、调模型、处理异步任务

系统设计能力
  -> 能拆模块、设计存储、做稳定性、考虑扩展性

AI 应用理解
  -> 知道 RAG / Prompt / Embedding / Agent 怎么工作

工程化意识
  -> 知道线上系统慢、贵、不稳定、不安全时怎么处理

面试表达能力
  -> 能用业务背景、架构、难点、指标讲项目
```

## 3. 面试题类型

### 类型 1：基础八股

常见范围：

- 网络：TCP、HTTP、HTTPS、SSE、WebSocket。
- OS：进程线程、锁、IO、多路复用。
- 数据库：索引、事务、锁、MVCC、SQL 优化。
- Redis：数据结构、缓存问题、分布式锁、持久化。
- MQ：可靠性、幂等、顺序、延迟队列。
- 分布式：限流、熔断、降级、服务治理。

回答标准：

```text
定义清楚
  -> 原理准确
  -> 能举业务例子
  -> 能说坑和取舍
```

### 类型 2：AI 应用题

常见范围：

- RAG 是什么？
- Embedding 是什么？
- 向量检索和关键词检索区别？
- Prompt 怎么设计？
- 如何让模型输出 JSON？
- Tool Calling 怎么保证安全？
- Agent 和 Workflow 区别？
- 如何降低模型延迟和成本？
- 如何评估 AI 应用效果？

回答标准：

```text
不要只背概念，要讲链路。
```

例如问 RAG：

```text
RAG 是检索增强生成。
离线阶段把文档解析、切分、向量化并写入向量库。
在线阶段把用户问题向量化，召回相关 chunk，经过权限过滤和重排后拼入 Prompt，再让大模型基于材料生成答案。
它能缓解模型知识过时和幻觉问题，但不能完全消除幻觉，所以还要做引用、拒答和评测。
```

### 类型 3：系统设计题

高频题：

- 设计企业知识库问答系统。
- 设计智能客服系统。
- 设计文档解析和摘要系统。
- 设计一个模型网关。
- 设计一个支持流式输出的聊天服务。
- 设计 RAG 评测平台。

回答框架：

```text
需求澄清
  -> 核心链路
  -> 模块拆分
  -> 存储设计
  -> 高并发与稳定性
  -> 安全权限
  -> 评测监控
  -> 扩展优化
```

### 类型 4：项目追问

面试官会问：

- 这个项目真实难点是什么？
- 你的 RAG 准确率如何提升？
- 为什么要用 MQ？
- 为什么要用 Redis？
- 向量库和 MySQL 怎么配合？
- 你如何做权限过滤？
- 模型调用失败怎么办？
- 你如何证明效果变好了？
- 如果 QPS 上来怎么扩展？

项目回答要有“工程味”：

```text
不是：我调用模型生成答案。
而是：我设计了文档入库、检索召回、模型生成、流式返回、日志评测、权限过滤和降级兜底的完整链路。
```

## 4. 你要形成的脑内地图

```text
用户问题
  -> 后端 API
  -> 鉴权限流
  -> 会话管理
  -> 判断是否需要检索 / 工具调用
  -> 检索知识库
  -> 重排材料
  -> 组装 Prompt
  -> 调模型网关
  -> 流式输出
  -> 保存日志
  -> 评测反馈
```

## 5. 秋招表达关键词

你要反复使用这些关键词：

- 全链路。
- 异步化。
- 幂等。
- 权限过滤。
- 元数据过滤。
- 混合检索。
- 重排。
- 流式响应。
- 超时重试。
- 熔断降级。
- Prompt 版本管理。
- 回归评测。
- 成本监控。
- 可观测性。
