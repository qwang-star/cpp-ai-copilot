# 00 总览与学习路线

## 1. 岗位定义

后端 AI 应用开发不是“训练大模型”的岗位，也不是“只调 API 写个聊天框”的岗位。

它的核心是：

```text
用后端工程能力，把模型能力接入业务系统，并让它稳定、可控、可评估、可扩展。
```

你要具备三层能力：

```text
后端系统能力
  -> 网络、数据库、缓存、消息队列、分布式、接口设计、稳定性

AI 应用能力
  -> LLM、Prompt、Embedding、RAG、Tool Calling、Agent、Workflow

工程化落地能力
  -> 权限、安全、评测、监控、成本、灰度、项目表达
```

## 2. 总体学习顺序

## 4. 面试官的真实关注点

面试官通常不会只问“RAG 是什么”，而会追问：

- 为什么这个系统需要 RAG？
- 文档怎么切？
- 检索不准怎么办？
- 大模型返回慢怎么办？
- 生成内容胡说怎么办？
- 用户没有权限的文档会不会被搜出来？
- 第三方模型挂了怎么办？
- 怎么知道改了 Prompt 以后效果变好了？
- 成本怎么控？
- 你这个项目和 Demo 有什么区别？

所以复习时要用这个公式：

```text
概念
  -> 原理
  -> 工程链路
  -> 常见问题
  -> 解决方案
  -> 项目表达
  -> 面试追问
```

## 6. 复习优先级

### P0：必须拿下

- HTTP / SSE / WebSocket。
- MySQL 索引、事务、锁、MVCC。
- Redis 缓存、限流、分布式锁。
- MQ 异步任务、重试、幂等。
- RAG 离线链路和在线链路。
- Embedding 与向量检索。
- Prompt 模板、结构化输出。
- Tool Calling 权限与参数校验。
- 模型调用超时、重试、降级。
- 项目架构讲解。

### P1：重点掌握

- Rerank、混合检索、Query Rewrite。
- Agent 与 Workflow 的取舍。
- 模型网关。
- LLMOps。
- Prompt 注入与越权检索。
- 评测集和回归测试。

### P2：了解即可

- 大模型训练细节。
- RLHF 数学细节。
- CUDA 推理优化。
- 向量索引底层实现。
- 模型微调全流程。
