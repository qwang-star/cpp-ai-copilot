# 02 完整性审查与改进记录

## 1. 审查结论

当前 `6_AI应用开发` 已经从“单篇总纲”整理成“按秋招复习顺序组织的专题知识库”。

整体结构符合后端 AI 应用开发岗位的准备路径：

```text
岗位认知
  -> 后端基础
  -> AI 基础
  -> RAG / Tool / Agent 应用链路
  -> 工程化
  -> 项目面试
```

## 2. 原先主要问题

### 问题 1：只有总架构，复习颗粒度太粗

单篇总纲适合建立全局认知，但不适合连续两个月复习。

已改进：

- 拆成 `00_导航`、`01_后端基础`、`02_AI核心`、`03_应用编排`、`04_工程化`、`05_项目面试`、`99_参考资料`。
- 每个文件夹对应一个复习阶段。

### 问题 2：AI 内容和后端基础容易割裂

很多资料会只讲 RAG、Prompt、Agent，但秋招面试仍然会追问 MySQL、Redis、MQ、网络、接口设计。

已改进：

- 在 `01_后端基础/00_后端基础与AI场景映射.md` 中把后端八股映射到 AI 场景。
- 补充了 `01_后端基础/01_编程语言_Web框架_API设计.md`，覆盖 Java / Python / Go、Web 框架、API、幂等、同步异步流式接口。

### 问题 3：缺少文件解析、多模态这类真实项目常见内容

RAG 项目里，文档解析质量往往决定效果上限。

已改进：

- 新增 `02_AI核心/03_文件解析_OCR_多模态基础.md`。
- 覆盖 PDF、Word、Markdown、HTML、Excel、OCR、多模态 RAG。

### 问题 4：缺少部署和推理服务视角

后端 AI 应用开发不一定要训练模型，但要知道模型服务如何部署、调用和扩容。

已改进：

- 新增 `04_工程化/03_部署_推理服务_云原生基础.md`。
- 覆盖第三方 API、自部署模型、推理服务、性能指标、容器化、Kubernetes、灰度回滚。

### 问题 5：面试输出材料不够集中

知识点会了，不代表能在面试中讲出来。

已改进：

- `05_项目面试/00_项目包装与面试题.md` 负责项目包装。
- `05_项目面试/01_两个月冲刺计划.md` 负责时间安排。
- `05_项目面试/02_AI应用高频面试题库.md` 负责高频问答。
