## 1. 覆盖清单

| 技术 | 旧技术/旧痛点 | 新技术改进 | 优点 | 缺点/代价 | 项目用法 | 详细文档 |
|---|---|---|---|---|---|---|
| QUIC v1 | TCP 队头阻塞、移动网络连接迁移差 | 基于 UDP 在用户态实现可靠传输、多路复用、TLS 1.3、Connection ID | 弱网更友好，建连更快，连接迁移好 | UDP 可达性、排查复杂、基础设施要求高 | 移动端 AI Copilot 入口协议演进 | [G01](G01_网络协议_QUIC_HTTP3_WebTransport.md) |
| HTTP/3 | HTTP/2 仍跑在 TCP 上 | HTTP 语义映射到 QUIC | 减少 TCP 层队头阻塞影响 | 网关/CDN/代理支持成本 | 网关层支持 HTTP/3，保留 HTTP/2 fallback | [G01](G01_网络协议_QUIC_HTTP3_WebTransport.md) |
| WebTransport | WebSocket 只提供可靠双向通信 | 支持可靠 stream 和不可靠 datagram | 低延迟，多模态实时场景更灵活 | 生态较新，服务端支持要求高 | 未来语音/多模态实时交互 | [G01](G01_网络协议_QUIC_HTTP3_WebTransport.md) |
| io_uring | epoll/系统调用在高性能 IO 下仍有切换成本 | SQ/CQ 环形队列组织异步 IO | 减少系统调用，批量提交，适合高吞吐 | Linux 相关，复杂度高 | 大量文件上传、日志、索引 IO 优化 | [G02](G02_高性能后端_io_uring_eBPF_OpenTelemetry.md) |
| eBPF | 应用日志看不到内核/网络细节 | 安全地把程序挂到内核事件上观测 | 不改业务代码，内核级可观测 | 权限高，学习曲线陡 | 排查 TCP 重传、系统调用慢、容器网络 | [G02](G02_高性能后端_io_uring_eBPF_OpenTelemetry.md) |
| OpenTelemetry | logs/metrics/traces 分散 | 统一 traces、metrics、logs 标准 | 跨服务链路清楚，厂商中立 | 接入和采样成本 | 追踪一次 AI 问答从 RAG 到模型的全链路 | [G02](G02_高性能后端_io_uring_eBPF_OpenTelemetry.md) |
| Hybrid Search | 纯向量检索漏精确词 | dense + sparse/BM25 融合 | 语义和关键词都兼顾 | 融合权重和评测复杂 | 企业制度金额、条款、编号检索 | [G03](G03_AI应用前沿_HybridRAG_GraphRAG_MCP_StructuredOutputs.md) |
| GraphRAG | 普通 chunk RAG 不擅长跨文档关系 | 抽取实体关系，图上检索和总结 | 适合全局总结和关系推理 | 构建/更新/评测成本高 | 制度流程、部门角色、文档冲突分析 | [G03](G03_AI应用前沿_HybridRAG_GraphRAG_MCP_StructuredOutputs.md) |
| Structured Outputs | “请输出 JSON”不稳定 | 用 schema 约束输出结构 | 后端解析稳定，适合工具参数 | 模型支持差异，后端仍需校验 | Tool Calling、引用、评测结果结构化 | [G03](G03_AI应用前沿_HybridRAG_GraphRAG_MCP_StructuredOutputs.md) |
| Prompt Caching | 长 Prompt 每次重复计算 | 复用稳定前缀 | 降低延迟和成本 | 命中依赖前缀稳定 | 固定 system prompt、工具 schema、输出格式 | [G03](G03_AI应用前沿_HybridRAG_GraphRAG_MCP_StructuredOutputs.md) |
| MCP | 工具接入方式不统一 | 标准化工具、资源、上下文接入 | 工具生态复用更容易 | 安全仍要自己做 | 后续把报销/请假/知识库工具标准化 | [G03](G03_AI应用前沿_HybridRAG_GraphRAG_MCP_StructuredOutputs.md)、[G06](G06_Agent工程化_AgentsSDK_A2A_MCP_Guardrails.md) |
| C++20 Coroutine | 回调式异步代码难读 | 异步流程写得像同步 | 可读性好，适合 IO 链路 | 语义复杂，依赖库支持 | RAG/模型/工具调用异步编排 | [G04](G04_Cpp现代后端生态_Coroutines_Asio_Drogon_uWebSockets.md) |
| Boost.Asio | 裸 socket 复杂 | 统一异步 IO、事件循环、timer | 成熟、跨平台、底层强 | 学习曲线陡 | 模型网关、自定义代理、网络层能力 | [G04](G04_Cpp现代后端生态_Coroutines_Asio_Drogon_uWebSockets.md) |
| Drogon | 手写 Web 服务效率低 | C++ 完整 Web 框架 | 路由/JSON/异步/数据库支持 | 生态不如 Java/Go 普及 | C++ AI Copilot API Server | [G04](G04_Cpp现代后端生态_Coroutines_Asio_Drogon_uWebSockets.md) |

| Gateway API | Ingress 表达有限、注解不统一 | Kubernetes 更现代的网关 API | 职责清楚，表达能力强 | 生态和实现依赖 | 多服务入口流量治理 | [G05](G05_云原生服务治理_GatewayAPI_Envoy_Istio_OpenFeature.md) |

| Agents SDK | 手写 Agent loop 混乱 | 统一 Agent、Tool、Handoff、Guardrails、Tracing | 工程结构清楚 | 框架变化快 | 学习 Agent 工程化思想 | [G06](G06_Agent工程化_AgentsSDK_A2A_MCP_Guardrails.md) |
| A2A | 多 Agent 协作协议不统一 | 标准化 Agent 间任务和消息 | 多 Agent 互操作 | 简单项目不需要 | 后续多业务 Agent 协作 | [G06](G06_Agent工程化_AgentsSDK_A2A_MCP_Guardrails.md) |
| Guardrails | 只靠 Prompt 防风险 | 输入、工具、输出、预算多层护栏 | Agent 更可控 | 需要系统设计 | 高风险工具人工确认、参数鉴权 | [G06](G06_Agent工程化_AgentsSDK_A2A_MCP_Guardrails.md) |
| OWASP LLM Top 10 | 传统 Web 安全不覆盖 LLM 风险 | LLM 应用威胁清单 | 风险系统化 | 需要结合业务落地 | Prompt 注入、越权、过度代理、成本风险 | [G07](G07_AI安全与供应链_OWASP_NIST_SLSA_Sigstore.md) |

## 2. 结论

这批前沿技术已经覆盖：

```text
网络协议：QUIC、HTTP/3、WebTransport
后端性能：io_uring、eBPF、OpenTelemetry
C++ 生态：Coroutine、Boost.Asio、Drogon、uWebSockets
RAG 前沿：Hybrid Search、GraphRAG
模型输出和成本：Structured Outputs、Prompt Caching
Agent 工程：MCP、Agents SDK、A2A、Guardrails
云原生治理：Gateway API、Envoy、Istio Ambient Mesh、OpenFeature
安全供应链：OWASP LLM Top 10、NIST AI RMF、SLSA、Sigstore
```

后续如果继续扩展，可以再补：

```text
多模态模型工程化
向量数据库性能调优实战
GPU 推理服务前沿
边缘 AI 部署
隐私计算和企业数据治理
```

## 3. 第二轮补充审计

|---|---|---|---|---|---|---|

| RRF | dense 和 BM25 分数尺度不同，难融合 | 用排名融合而不是原始分数硬加权 | 简单稳健，适合混合检索第一版 | 不利用原始分数强弱 | 向量召回和关键词召回合并 | [G08](G08_向量检索前沿_VectorDB_HNSW_Quantization_LateInteraction.md) |
| Late Interaction / ColBERT | 单向量 chunk 压缩太狠，细节丢失 | token 级向量细粒度匹配 | 代码、法律、条款检索更准 | 存储和计算更重 | 高价值知识库或代码问答增强 | [G08](G08_向量检索前沿_VectorDB_HNSW_Quantization_LateInteraction.md) |
