# G99 前沿技术覆盖矩阵

这个文件用来快速查漏。

目标不是“追所有热点”，而是覆盖后端 AI 应用开发最容易被问到的新技术方向：

```text
网络协议
C++ 现代后端
高性能 IO 和可观测性
RAG 和 Agent
云原生服务治理
安全合规和供应链
向量检索和推理服务
实时多模态和端侧 AI
数据治理和隐私
```

## 1. 网络协议

| 技术 | 解决旧技术什么问题 | 主要代价 | 去哪里看 |
|---|---|---|---|
| QUIC v1 | TCP 队头阻塞、建连延迟、移动网络连接迁移差 | UDP 可达性、排查复杂、基础设施支持 | [G01](G01_网络协议_QUIC_HTTP3_WebTransport.md) |
| HTTP/3 | HTTP/2 仍受 TCP 层队头阻塞影响 | 网关/CDN/代理支持成本 | [G01](G01_网络协议_QUIC_HTTP3_WebTransport.md) |
| TLS 1.3 | TLS 1.2 握手往返更多、旧算法复杂 | 0-RTT 有重放风险 | [G01](G01_网络协议_QUIC_HTTP3_WebTransport.md) |
| WebTransport | WebSocket 只有可靠双向流，不适合部分可丢的低延迟数据 | 生态更新、服务端支持要求高 | [G01](G01_网络协议_QUIC_HTTP3_WebTransport.md) |

## 12. 面试总回答模板

```text
我理解新技术时会先看它解决了旧方案什么瓶颈。比如网络层，HTTP/3 不是改变 HTTP 语义，而是用 QUIC 替换 TCP 作为传输基础，改善 HTTP/2 仍受 TCP 队头阻塞影响的问题。C++ 后端层，我会用 Drogon 这类框架承载 Web API，用 C++20 coroutine 和 Boost.Asio 理解异步 IO。AI 应用层，我会从纯向量 RAG 演进到 Hybrid Search、RRF、Rerank，必要时评估 Late Interaction；推理服务层，如果从第三方 API 走向私有化，会用模型网关接 vLLM、Triton 或 TensorRT-LLM，并关注 PagedAttention、Continuous Batching、KV Cache、TTFT 和 tokens/s。实时交互上，文本流用 SSE，语音和多模态实时会话再评估 WebRTC、Realtime API 和 WebTransport。安全治理上，会做 PII 脱敏、ABAC/ReBAC 权限、RAG metadata filter、审计和删除链路，同时参考 OWASP LLM Top 10、NIST AI RMF、SLSA、Sigstore。
```
