# C01 机器学习基础

## 1. 机器学习是什么

### 是什么

机器学习是让计算机从数据中学习规律，并用这些规律对新数据做预测或决策的方法。

传统编程是：

```text
规则 + 数据 -> 结果
```

机器学习是：

```text
数据 + 结果 -> 学到规则
学到的规则 + 新数据 -> 新结果
```

### 为什么重要

后端 AI 应用开发不一定训练模型，但要理解模型能力、指标和评测，否则很难判断模型效果。

### 后端 AI 应用场景

- 判断用户问题类别。
- 判断答案是否相关。
- 判断内容是否违规。
- 预测用户是否满意。
- 检索结果排序。

## 2. 监督学习

### 是什么

监督学习是用带标签的数据训练模型。

例子：

```text
输入：用户问题
标签：售前 / 售后 / 投诉 / 技术支持
```

模型学习后，可以给新问题分类。

### 适合任务

- 分类。
- 回归。
- 信息抽取。
- 排序。

### AI 场景

智能客服中，先用分类模型判断用户意图，再决定走知识库问答、订单查询还是转人工。

## 3. 无监督学习

### 是什么

无监督学习使用没有标签的数据，让模型自己发现结构。

常见任务：

- 聚类。
- 降维。
- 异常检测。

### AI 场景

- 把相似用户问题聚类，发现高频问题。
- 把相似文档聚类，辅助知识库整理。
- 从日志中发现异常调用行为。

## 4. 强化学习

### 是什么

强化学习是智能体通过和环境交互，根据奖励信号学习策略。

核心元素：

- Agent。
- Environment。
- State。
- Action。
- Reward。

### 大模型关系

RLHF 使用人类反馈训练奖励模型，再优化大模型行为，让模型更符合人类偏好。

### 秋招掌握程度

了解概念即可，不需要深挖公式。

## 5. 分类

### 是什么

分类是预测离散类别。

例子：

```text
用户问题 -> 技术问题 / 费用问题 / 账号问题
```

### 常见指标

- Accuracy。
- Precision。
- Recall。
- F1。

### AI 场景

- 意图识别。
- 内容安全分类。
- 文档类型判断。
- 工单分类。

## 6. 回归

### 是什么

回归是预测连续数值。

例子：

```text
预测一次请求的成本
预测用户满意度分数
预测任务耗时
```

### 指标

- MSE。
- MAE。
- RMSE。

## 7. 聚类

### 是什么

聚类是把相似样本分到同一组。

常见算法：

- K-Means。
- DBSCAN。
- 层次聚类。

### AI 场景

- 用户问题聚类。
- Bad case 聚类。
- 文档主题聚类。
- 相似 FAQ 合并。

## 8. 训练集、验证集、测试集

### 是什么

数据通常拆成：

- 训练集：训练模型。
- 验证集：调参数、选模型。
- 测试集：最终评估。

### 为什么不能混

如果测试集参与训练或调参，评估结果会虚高。

### AI 应用场景

RAG 评测集也要固定，不能每次优化时只挑有利样本。

## 9. 过拟合

### 是什么

模型在训练集表现很好，但在新数据上表现差。

直观理解：

```text
模型背答案了，而不是学规律。
```

### 原因

- 模型太复杂。
- 数据太少。
- 噪声太多。
- 训练太久。

### 解决

- 增加数据。
- 正则化。
- Dropout。
- 早停。
- 数据增强。

### AI 场景

Prompt 或 RAG 策略如果只针对少量样例优化，也可能在其他问题上退化，这是一种应用层面的“过拟合”。

## 10. 欠拟合

### 是什么

模型在训练集和测试集上表现都差，说明模型没学到规律。

### 原因

- 模型太简单。
- 特征不足。
- 训练不够。
- 数据质量差。

## 11. 正则化

### 是什么

正则化是在损失函数中加入约束，防止模型过度复杂。

常见：

- L1。
- L2。

### 直观理解

让模型不要为了拟合训练数据而把参数变得过于极端。

## 12. 梯度下降

### 是什么

梯度下降是优化模型参数的方法。

核心思想：

```text
沿着损失函数下降最快的方向，一步步调整参数。
```

### 常见变体

- Batch Gradient Descent。
- SGD。
- Mini-batch SGD。
- Adam。

## 13. 损失函数

### 是什么

损失函数衡量模型预测和真实答案之间的差距。

常见：

- 交叉熵：分类。
- MSE：回归。

### AI 场景

大模型训练也依赖损失函数，本质上是在预测下一个 token 时最小化预测分布和真实 token 的差距。

## 14. Accuracy

### 是什么

准确率 = 预测正确数量 / 总数量。

### 局限

类别不均衡时会误导。

例子：

```text
1000 条样本里 990 条正常，10 条违规。
模型全部预测正常，Accuracy = 99%，但完全抓不住违规。
```

## 15. Precision

### 是什么

精确率表示模型预测为正的样本中，有多少是真的正。

```text
Precision = TP / (TP + FP)
```

### AI 场景

内容安全中，高 Precision 意味着被判违规的内容大多真的违规，误杀少。

## 16. Recall

### 是什么

召回率表示所有真实正样本中，有多少被模型找到了。

```text
Recall = TP / (TP + FN)
```

### AI 场景

RAG 里的 Recall@K 表示正确资料是否被召回。召回不到，模型就很难回答正确。

## 17. F1

### 是什么

F1 是 Precision 和 Recall 的调和平均。

```text
F1 = 2 * P * R / (P + R)
```

### 适用

类别不均衡、需要兼顾误报和漏报的场景。

## 18. AUC

### 是什么

AUC 衡量模型把正样本排在负样本前面的能力。

### AI 场景

用于分类、风控、排序评估等。

## 19. 特征工程

### 是什么

特征工程是把原始数据转成模型更容易学习的表示。

传统机器学习很依赖特征工程。

### 大模型时代变化

大模型和深度学习可以自动学习表示，但应用工程仍然需要：

- 数据清洗。
- 字段结构化。
- metadata 设计。
- 文档切分。

这些可以看作 AI 应用层面的特征工程。

## 20. 面试回答模板

```text
机器学习的核心是从数据中学习规律。后端 AI 应用开发不一定训练模型，但要理解分类、召回、过拟合和评测指标。比如 RAG 里检索阶段关注 Recall@K，生成阶段关注答案正确性和忠实性；如果只看 Accuracy 或用户点赞，很容易误判系统效果。
```
