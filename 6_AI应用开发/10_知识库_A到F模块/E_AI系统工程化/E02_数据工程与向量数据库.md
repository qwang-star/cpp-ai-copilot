# E02 数据工程与向量数据库

## 1. 数据工程是什么

### 是什么

数据工程是把原始数据加工成 AI 系统可用数据的过程。

RAG 中包括：

```text
采集 -> 解析 -> 清洗 -> 切分 -> 标注 metadata -> 向量化 -> 索引 -> 质量检查
```

## 2. 数据质量

### 为什么重要

RAG 的上限常常由数据质量决定。

坏数据会导致：

- 召回不到。
- 召回错误。
- 答案幻觉。
- 引用错误。

## 3. 数据清洗

### 是什么

去除噪声并保留结构。

包括：

- 去重复。
- 去页眉页脚。
- 修复换行。
- 表格结构化。
- 保留标题层级。
- 删除乱码。

## 4. 去重

### 是什么

识别重复文档或重复 chunk。

方法：

- 文件 hash。
- 文本 hash。
- SimHash。
- Embedding 相似度。

### AI 场景

重复文档会导致召回结果被同一内容刷屏。

## 5. Metadata

### 是什么

metadata 是数据的业务属性。

字段：

- tenant_id。
- document_id。
- chunk_id。
- page。
- section。
- owner。
- acl。
- created_at。

### 用途

- 权限过滤。
- 引用来源。
- 删除索引。
- 版本管理。
- 数据分析。

## 6. 向量数据库是什么

### 是什么

向量数据库用于存储向量，并支持相似度检索。

### 组成

- collection。
- vector。
- payload / metadata。
- index。
- search。
- filter。

## 7. Collection

### 是什么

Collection 是一组向量数据的集合。

### AI 场景

可以按租户、业务线、embedding 模型版本建立 collection。

## 8. Upsert

### 是什么

插入或更新向量。

### AI 场景

文档更新后，重新生成 chunk 和 embedding，需要 upsert 新向量。

## 9. Delete

### 是什么

删除向量记录。

### AI 场景

文档删除时，必须删除对应 chunk 向量，否则可能检索到已删除内容。

## 10. HNSW

### 是什么

HNSW 是常见近似最近邻索引，基于多层图结构。

### 直观理解

像高速公路 + 普通道路：

- 高层图快速接近目标区域。
- 底层图精细搜索。

### 掌握程度

知道它用于快速向量检索即可。

## 11. IVF

### 是什么

IVF 先把向量聚类分桶，查询时只搜索相关桶。

### 特点

速度快，但可能损失召回。

## 12. PQ

### 是什么

PQ 是向量压缩技术，降低存储和内存。

### 代价

可能损失精度。

## 13. 向量库和 MySQL 的关系

### 分工

```text
MySQL：业务数据、事务、权限、状态
向量库：语义检索
对象存储：原始文件
Redis：缓存和限流
```

### AI 场景

向量库保存 chunk_id，查到后回 MySQL 获取文档和权限信息。

## 14. Embedding 模型升级

### 问题

不同模型的向量空间不同，不能混用。

### 处理

- 新建 collection。
- 双写新旧向量。
- 灰度查询。
- 完成后切流量。

## 15. 数据版本管理

### 需要版本化

- 文档版本。
- chunk 版本。
- embedding 模型版本。
- 索引版本。
- Prompt 版本。

## 16. 面试回答模板

```text
RAG 的数据工程包括文档解析、清洗、切分、metadata 标注、向量化和索引。向量库负责语义检索，MySQL 负责业务数据和权限。文档删除或更新时，要同步更新向量索引；如果 embedding 模型升级，通常要重建索引，因为新旧向量空间不能混用。
```

## 17. 前沿升级：向量检索调优、量化、Late Interaction

当知识库从几千个 chunk 增长到百万级、千万级 chunk 时，不能只说“用向量数据库”。

要继续追问：

```text
召回准不准？
P95 延迟高不高？
内存成本贵不贵？
权限 filter 会不会拖慢？
金额、条款号、代码符号会不会漏？
```

现代做法通常是：

```text
HNSW 调 efSearch：在召回和延迟之间平衡
Hybrid Search：dense 语义 + sparse/BM25 关键词
RRF：融合不同检索器的排名
Rerank：把 Top50 精排成 Top5
PQ/SQ/Binary Quantization：降低向量存储和内存成本
Late Interaction / ColBERT：高价值场景做 token 级细粒度匹配
```

项目里推荐路线：

```text
第一版：向量检索 + metadata filter
第二版：Hybrid Search + Rerank
第三版：HNSW 参数压测和评测集
第四版：冷数据量化
第五版：高价值知识库评估 Late Interaction
```
