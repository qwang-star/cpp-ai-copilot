# A02 操作系统

## 1. 进程

### 是什么

进程是操作系统进行资源分配的基本单位。

一个进程拥有：

- 独立地址空间。
- 文件描述符。
- 堆。
- 栈。
- 代码段。
- 数据段。

### 为什么重要

后端服务通常以进程形式运行。多个服务实例本质就是多个进程。

### AI 场景

- Web 服务进程处理用户请求。
- Worker 进程处理文档解析和 Embedding。
- 模型推理服务进程加载大模型。

### 面试回答

```text
进程是资源分配单位，线程是 CPU 调度单位。进程之间地址空间隔离，稳定性更好，但通信成本更高。
```

## 2. 线程

### 是什么

线程是 CPU 调度的基本单位。同一进程内线程共享进程资源。

共享：

- 堆。
- 全局变量。
- 文件描述符。

独有：

- 栈。
- 程序计数器。
- 寄存器上下文。

### AI 场景

- 后端线程处理 HTTP 请求。
- 线程池执行模型调用、文档解析、数据库访问。
- 多线程批量生成 Embedding。

### 常见坑

- 线程过多导致上下文切换开销。
- 共享变量导致线程安全问题。
- 阻塞调用占满线程池。

## 3. 协程

### 是什么

协程是用户态的轻量级执行单元，由程序或运行时调度，不完全依赖操作系统线程调度。

### 优点

- 创建成本低。
- 切换成本低。
- 适合 IO 密集型任务。

### AI 场景

模型 API 调用、向量库查询、HTTP 请求都是 IO 密集型，适合异步协程提高并发。

### 面试回答

```text
线程由操作系统调度，协程通常由用户态运行时调度。协程更轻量，适合大量 IO 等待场景，但 CPU 密集任务仍然需要多线程或多进程。
```

## 4. 上下文切换

### 是什么

上下文切换是 CPU 从一个任务切换到另一个任务时，保存和恢复执行现场的过程。

保存内容：

- 寄存器。
- 程序计数器。
- 栈指针。
- 内存映射信息。

### 为什么有开销

- 保存和恢复上下文需要时间。
- CPU 缓存可能失效。
- 线程过多会频繁切换。

### AI 场景

如果模型调用使用大量阻塞线程，线程数暴涨，会导致上下文切换和内存开销增加。

## 5. 用户态与内核态

### 是什么

操作系统为了安全，把运行状态分为用户态和内核态。

- 用户态：普通应用程序运行。
- 内核态：操作系统内核运行，可以访问硬件和核心资源。

### 切换场景

- 系统调用。
- 文件读写。
- 网络 IO。
- 进程调度。

### AI 场景

网络请求、文件上传、磁盘读写都涉及系统调用。大量小文件读写和频繁网络 IO 会增加系统调用开销。

## 6. 锁

### 是什么

锁用于保护共享资源，避免并发读写导致数据不一致。

常见锁：

- 互斥锁。
- 读写锁。
- 自旋锁。
- 乐观锁。
- 悲观锁。

### AI 场景

- 同一个文档不能被重复索引。
- 同一个用户额度扣减要并发安全。
- 同一个会话写消息要保证顺序。

### 常见坑

- 锁粒度太大影响性能。
- 忘记释放锁。
- 分布式场景只用本地锁无效。

## 7. 死锁

### 是什么

死锁是多个线程互相等待对方持有的资源，导致都无法继续执行。

四个必要条件：

- 互斥。
- 占有且等待。
- 不可抢占。
- 循环等待。

### 如何避免

- 固定加锁顺序。
- 减少锁范围。
- 加锁超时。
- 避免嵌套锁。
- 使用无锁或乐观锁。

### 面试回答

```text
死锁产生需要互斥、占有且等待、不可抢占、循环等待四个条件。避免死锁可以破坏其中任意一个条件，比如固定加锁顺序、加锁超时、减少锁粒度。
```

## 8. 内存管理

### 是什么

操作系统负责管理进程内存，包括分配、回收、隔离和映射。

核心概念：

- 虚拟内存。
- 物理内存。
- 页。
- 页表。
- 缺页中断。
- 堆。
- 栈。

### AI 场景

- 上传大文件不能一次性读入内存。
- 文档解析可能产生大文本，需流式处理。
- 模型推理服务显存和内存压力大。

### 常见坑

- 一次性读取大 PDF。
- 批量 Embedding 时把所有 chunk 放内存。
- 日志或响应体过大导致 OOM。

## 9. 虚拟内存

### 是什么

虚拟内存让每个进程看起来拥有连续独立的地址空间，实际由操作系统映射到物理内存。

### 作用

- 进程隔离。
- 简化内存管理。
- 支持比物理内存更大的地址空间。
- 支持内存换页。

### AI 场景

大文件处理和模型推理都要关注内存占用。虚拟内存不是无限资源，频繁换页会导致性能急剧下降。

## 10. 文件系统

### 是什么

文件系统负责组织和管理磁盘上的文件。

关注点：

- 顺序读写。
- 随机读写。
- 缓冲区。
- 文件描述符。
- Page Cache。

### AI 场景

- 文档上传。
- OCR 图片临时文件。
- 模型文件加载。
- 日志写入。

### 工程建议

- 大文件流式读写。
- 原文件放对象存储。
- 本地临时文件及时清理。
- 限制上传大小和类型。

## 11. 阻塞 IO

### 是什么

调用 IO 操作后，线程会一直等待结果返回。

### 优点

- 编程简单。

### 缺点

- 并发高时线程容易被占满。

### AI 场景

同步调用模型 API 如果阻塞时间很长，会占住请求线程。

## 12. 非阻塞 IO

### 是什么

IO 没准备好时立即返回，不阻塞线程。

### 特点

- 需要不断轮询。
- 单独使用会浪费 CPU。

## 13. IO 多路复用

### 是什么

一个线程可以同时监听多个 IO 事件。

常见机制：

- select。
- poll。
- epoll。

### 为什么重要

高并发网络服务常用 IO 多路复用减少线程数量。

### AI 场景

支持大量 SSE 长连接时，事件驱动模型比一个连接一个线程更可扩展。

## 14. 异步 IO

### 是什么

发起 IO 后立即返回，IO 完成后由系统通知应用。

### AI 场景

适合大量网络 IO，比如异步调用多个检索服务、模型服务、工具服务。

## 15. 线程池

### 是什么

线程池复用一组线程执行任务，避免频繁创建销毁线程。

核心参数：

- 核心线程数。
- 最大线程数。
- 队列大小。
- 空闲线程存活时间。
- 拒绝策略。

### AI 场景

- 文档解析线程池。
- Embedding 任务线程池。
- 模型调用线程池。
- 文件上传处理线程池。

### 常见坑

- 无界队列导致内存膨胀。
- 线程池过小导致任务堆积。
- 线程池过大导致切换开销。
- CPU 密集和 IO 密集任务共用线程池。

### 面试回答

```text
线程池要根据任务类型配置。CPU 密集型线程数接近 CPU 核数，IO 密集型可以适当更多。AI 应用里模型调用和向量库查询是 IO 密集，文档解析和 OCR 可能偏 CPU 密集，最好隔离线程池。
```
