# D05 多轮记忆与多模态

## 1. 多轮对话是什么

### 是什么

多轮对话是模型基于历史上下文持续和用户交流。

### AI 场景

用户：

```text
报销流程是什么？
需要哪些材料？
那多久到账？
```

后两个问题依赖前文。

## 2. Conversation

### 是什么

Conversation 是一次对话会话。

典型字段：

```text
conversation_id
user_id
tenant_id
title
created_at
updated_at
```

## 3. Message

### 是什么

Message 是会话中的一条消息。

字段：

```text
message_id
conversation_id
role
content
token_count
created_at
```

## 4. 上下文管理

### 问题

历史消息不能无限塞进 Prompt。

原因：

- 上下文窗口有限。
- 成本增加。
- 延迟增加。
- 噪声增加。

## 5. 历史裁剪

### 是什么

只保留最近 N 轮对话。

### 优点

简单。

### 缺点

早期重要信息可能丢失。

## 6. 对话摘要

### 是什么

把早期历史压缩成摘要。

策略：

```text
最近 N 轮原文 + 早期摘要
```

### 常见坑

摘要可能丢细节或引入错误。

## 7. 长期记忆

### 是什么

把用户偏好、长期事实、历史信息存储起来，后续检索使用。

### AI 场景

- 用户偏好。
- 常用项目。
- 常问问题。

### 安全

长期记忆可能涉及隐私，必须支持查看和删除。

## 8. 记忆检索

### 是什么

把历史记忆向量化，需要时按当前问题检索相关记忆。

### 链路

```text
用户问题
  -> 检索相关长期记忆
  -> 拼入 Prompt
```

## 9. 多模态是什么

### 是什么

多模态是处理文本、图片、音频、视频等多种数据形式。

### 常见能力

- 图片理解。
- OCR。
- 语音转文字。
- 文字转语音。
- 视频摘要。
- 文档截图问答。

## 10. OCR

### 是什么

OCR 是从图片或扫描件中识别文字。

### AI 场景

扫描 PDF、合同图片、发票、表格都需要 OCR。

## 11. 图片理解

### 是什么

模型理解图片内容并回答问题。

### AI 场景

- 图表解读。
- 截图问答。
- 商品识别。
- 工单图片分析。

## 12. ASR

### 是什么

ASR 是 Automatic Speech Recognition，语音转文字。

### AI 场景

- 语音客服。
- 会议记录。
- 面试录音转写。

## 13. TTS

### 是什么

TTS 是 Text To Speech，文字转语音。

### AI 场景

- 语音助手。
- 客服播报。
- 无障碍阅读。

## 14. 多模态 RAG

### 是什么

把图片、表格、扫描件中的信息也纳入检索增强生成。

常见链路：

```text
文件
  -> OCR / 图片描述 / 表格解析
  -> 文本化
  -> embedding
  -> 向量检索
  -> 多模态模型生成
```

## 15. 文件存储

### AI 场景

多模态文件通常不能直接存数据库。

建议：

- 原文件存对象存储。
- MySQL 存 metadata。
- 向量库存 chunk。
- 解析产物单独存储。

## 16. 面试回答模板

```text
多轮对话的核心是上下文管理，不能无限保存历史消息到 Prompt。我会采用最近 N 轮原文加早期摘要，并对长期记忆做向量检索。多模态应用中，图片和扫描件通常先经过 OCR 或视觉模型转成结构化信息，再进入 RAG 或业务流程。
```

## 17. 前沿升级：实时语音、多模态会话、端侧 AI

多模态不只是“能传图片”，还可以继续演进成实时交互：

```text
文本聊天：SSE 流式输出
任务状态：WebSocket 双向推送
实时语音：WebRTC / Realtime API
低延迟多模态数据：WebTransport
端侧预处理：WebGPU / WebNN / ONNX Runtime
本地小模型：llama.cpp / GGUF
```

旧做法：

```text
录完一整段音频
上传
ASR
LLM
TTS
再播放
```

新做法：

```text
边说边传
边识别
边生成
边播报
允许打断
```

项目里不要一开始就堆满复杂协议，推荐路线是：

```text
第一版：文本 + SSE
第二版：图片/OCR/表格解析
第三版：WebSocket 推 Agent 任务状态
第四版：语音上传 + ASR
第五版：WebRTC 或 Realtime API 做实时语音
第六版：端侧做图片压缩、敏感信息预检测、轻量分类
```
