# F03 系统设计题库

## 1. 设计企业知识库问答系统

### 回答结构

```text
需求澄清
  -> 文档类型、权限、QPS、延迟、是否引用

总体架构
  -> API、文档服务、RAG 服务、模型网关、存储

离线链路
  -> 上传、解析、切分、Embedding、入向量库

在线链路
  -> 提问、召回、重排、生成、引用

工程化
  -> 异步、缓存、限流、降级、评测、安全
```

## 2. 设计智能客服系统

### 核心模块

- 用户会话。
- 意图识别。
- FAQ / RAG。
- 订单工具。
- 工单工具。
- 转人工。
- 质检。

### 难点

- 多轮上下文。
- Tool Calling 权限。
- 转人工策略。
- 客服答案准确性。

## 3. 设计模型网关

### 核心模块

- 统一 API。
- 模型路由。
- 供应商适配。
- 超时重试。
- 熔断降级。
- 限流。
- 成本统计。
- 日志监控。

### 关键表

```text
model_config
model_route_rule
model_call_log
tenant_quota
prompt_version
```

## 4. 设计文档解析系统

### 核心链路

```text
上传
  -> 对象存储
  -> MQ
  -> 解析 Worker
  -> OCR / 表格解析
  -> 清洗
  -> 结构化 Markdown
  -> chunk
```

### 难点

- PDF 版式。
- 表格结构。
- 扫描件 OCR。
- 大文件。
- 失败重试。

## 5. 设计 RAG 评测平台

### 核心模块

- 评测集管理。
- 策略配置。
- 批量运行。
- 检索指标。
- 生成指标。
- LLM-as-a-Judge。
- 报告对比。

### 指标

- Recall@K。
- MRR。
- Answer Correctness。
- Faithfulness。
- Citation Accuracy。
- P95 latency。
- Cost。

## 6. 设计支持流式输出的聊天服务

### 链路

```text
客户端
  -> 建立 SSE
  -> 后端调用模型 stream
  -> 转发 delta
  -> 保存完整答案
  -> 发送 done
```

### 难点

- 客户端断开。
- 中途错误。
- 代理缓冲。
- 心跳。
- 完整消息落库。
