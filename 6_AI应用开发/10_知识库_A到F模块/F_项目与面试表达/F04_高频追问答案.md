# F04 高频追问答案

## 1. 你的项目和普通调用大模型 Demo 有什么区别？

我的项目不是只调用模型 API，而是围绕企业知识问答做了完整工程链路，包括文档上传、异步解析、chunk 切分、Embedding、向量检索、混合检索、Rerank、Prompt 生成、模型网关、流式输出、权限过滤、日志评测和降级兜底。

## 2. 为什么文档处理要异步？

文档解析、OCR、切分和 Embedding 都可能耗时较长。如果同步处理，会导致接口超时并占用请求线程。异步化后，上传接口只返回 task_id，后台 Worker 通过 MQ 处理任务，前端查询状态即可。

## 3. 为什么要用 MQ？

MQ 用于异步、解耦、削峰和失败重试。文档上传后发送解析消息，解析完成后发送向量化消息。Worker 可以独立扩容，失败消息可以重试，多次失败进入死信队列。

## 4. 如何保证 MQ 重复消费不出问题？

通过业务幂等保证。比如 embedding 任务使用 `document_id + chunk_id + embedding_version` 作为幂等 key，写向量库前检查是否已处理，MySQL 也可以加唯一约束。

## 5. 为什么用 Redis？

Redis 用于缓存热点问答和检索结果、用户级限流、token 额度计数、任务幂等、分布式锁和短期会话状态。

## 6. AI 问答结果能不能缓存？

可以，但要看场景。高频、确定性、权限一致的问题适合缓存；个性化、实时性强、权限敏感的问题不适合或要谨慎。缓存 key 应包含 user_id、kb_id、query_hash、prompt_version、model_version。

## 7. 为什么向量库不能替代 MySQL？

向量库负责语义检索，MySQL 负责业务数据、事务、权限、状态和日志。比如文档状态、用户权限、会话消息都更适合 MySQL。向量库里的 chunk_id 需要回到 MySQL 查询业务信息。

## 8. Chunk 太大或太小有什么问题？

chunk 太大，召回不精准、噪声多、占 token；chunk 太小，语义不完整、上下文断裂。实际会按标题、段落或语义切分，并加适当 overlap。

## 9. RAG 为什么不能完全消除幻觉？

RAG 只能提供外部依据，但检索可能错误，模型也可能忽略资料或过度推断。所以还要有 Prompt 约束、引用来源、拒答策略、答案校验和评测。

## 10. 如何提高 RAG 准确率？

从数据、检索、生成三层优化：数据层提升解析和 chunk 质量；检索层使用 Query Rewrite、混合检索、Rerank；生成层优化 Prompt、要求引用和拒答；最后用固定评测集验证效果。

## 11. 如何做权限过滤？

文档和 chunk 入库时写入 tenant_id、kb_id、owner_id、role、acl 等 metadata。检索时根据当前用户构造 filter，只召回有权限的 chunk，返回引用前再次校验，并记录审计日志。

## 12. 模型调用失败怎么办？

通过模型网关统一处理超时、重试、熔断和降级。可重试错误做指数退避；模型异常时切备用模型、返回缓存答案、返回检索片段或提示稍后重试。

## 13. 如何降低模型调用成本？

使用模型分级、缓存高频问题、Prompt 压缩、控制 TopK 和 max_tokens、批量 Embedding、限制重试次数，并按用户和租户统计 token 成本。

## 14. SSE 和 WebSocket 怎么选？

普通大模型文本流式输出用 SSE 更简单，因为是服务端单向推送。需要双向实时通信，比如语音对话或协作场景，再用 WebSocket。

## 15. 如何评估项目效果？

分三类指标：检索看 Recall@K、MRR；生成看答案正确率、忠实性、引用准确率、幻觉率；工程看 P95 延迟、错误率、首 token 延迟和单次请求成本。

## 16. Prompt 改了怎么保证不出问题？

Prompt 要版本管理。每次改动后，用 Golden Dataset 做回归测试，对比正确率、格式合法率、幻觉率、拒答准确率和 token 成本，再灰度上线。

## 17. Tool Calling 怎么保证安全？

模型只负责选择工具和生成参数。后端必须做工具白名单、JSON Schema 参数校验、用户鉴权、资源权限校验、业务规则校验。高风险操作需要人工确认。

## 18. Agent 为什么生产落地难？

Agent 自主规划带来不确定性，可能步数、成本、延迟失控，也更难调试和复现。生产系统通常用 Workflow 固定关键链路，只在局部任务中引入 Agent。

## 19. 如果用户问“总结整篇文档”，普通 RAG 为什么不够？

普通 TopK RAG 只召回局部片段，适合回答局部问题。总结整篇文档需要文档级处理，比如章节摘要、Map-Reduce 总结或分层摘要。
