# B04 认证鉴权

## 1. 认证和鉴权区别

### 认证 Authentication

确认你是谁。

例子：

- 登录。
- Token 校验。
- 手机验证码。

### 鉴权 Authorization

确认你能做什么。

例子：

- 能否访问某知识库。
- 能否删除文档。
- 能否调用退款工具。

### AI 场景

RAG 权限问题非常关键：用户只能检索自己有权限的文档。

## 2. Cookie / Session

### 是什么

Session 存在服务端，Cookie 保存 session_id。

### 优点

- 服务端可控。
- 易失效。

### 缺点

- 分布式下要共享 Session。
- 移动端和开放 API 不够方便。

## 3. JWT

### 是什么

JWT 是自包含 Token，包含用户信息和签名。

结构：

```text
Header.Payload.Signature
```

### 优点

- 无状态。
- 适合分布式。

### 缺点

- 失效控制不灵活。
- Payload 不应放敏感信息。
- Token 泄露风险。

### AI 场景

Chat Service、RAG Service、Model Gateway 可以从 JWT 中获取 user_id、tenant_id，再做权限过滤。

## 4. OAuth2

### 是什么

OAuth2 是授权协议，允许第三方应用在用户授权下访问资源。

常见角色：

- Resource Owner。
- Client。
- Authorization Server。
- Resource Server。

### AI 场景

企业 AI 助手接入飞书、企业微信、Google Drive、GitHub 时，常用 OAuth2 获取授权。

## 5. RBAC

### 是什么

RBAC 是基于角色的访问控制。

模型：

```text
User -> Role -> Permission
```

### AI 场景

角色：

- 普通员工。
- 知识库管理员。
- 部门主管。
- 系统管理员。

权限：

- 查看知识库。
- 上传文档。
- 删除文档。
- 邀请成员。
- 查看调用日志。

## 6. ABAC

### 是什么

ABAC 是基于属性的访问控制。

根据属性判断：

- 用户部门。
- 文档密级。
- 租户。
- 时间。
- 地域。

### AI 场景

RAG 检索时根据 metadata 做权限过滤：

```text
tenant_id = 当前租户
department_id in 用户部门
confidential_level <= 用户级别
```

## 7. 接口签名

### 是什么

接口签名用于验证请求没有被篡改，并确认调用方身份。

常见字段：

- app_id。
- timestamp。
- nonce。
- signature。

### AI 场景

第三方系统调用你的 AI 服务，或模型工具调用内部接口时，可以使用签名增强安全。

## 8. 防重放

### 是什么

防止攻击者复制旧请求再次发送。

方案：

- timestamp。
- nonce。
- 签名。
- nonce 缓存。

## 9. RAG 权限过滤

### 错误做法

```text
先搜所有文档，再让模型判断哪些能回答。
```

### 正确做法

```text
用户请求
  -> 认证 user_id / tenant_id
  -> 查询用户权限
  -> 构造向量库 metadata filter
  -> 只召回有权限 chunk
  -> 生成答案
  -> 引用返回前再次校验
```

## 10. Tool Calling 鉴权

### 原则

```text
模型不能决定权限，后端决定权限。
```

每次工具调用检查：

- 用户身份。
- 工具权限。
- 资源权限。
- 参数合法性。
- 操作风险。

高风险操作：

- 删除。
- 支付。
- 退款。
- 审批。
- 发邮件给外部。

需要人工确认或二次校验。

## 11. 面试回答模板

```text
认证解决用户是谁，鉴权解决用户能访问什么。AI 应用里尤其要注意 RAG 越权和 Tool Calling 越权。我会在检索前基于 tenant、role、ACL 构造 metadata filter，只召回用户有权限的 chunk；工具调用也必须经过后端鉴权和参数校验，不能把权限判断交给模型。
```
