# 02 AI 应用高频面试题库

## 1. RAG

### 什么是 RAG？

RAG 是检索增强生成。它先从外部知识库检索和用户问题相关的资料，再把资料作为上下文交给大模型生成答案。它适合企业私有知识问答、知识实时更新和需要引用来源的场景。

### RAG 离线和在线链路是什么？

离线链路：

```text
文档上传 -> 解析 -> 清洗 -> 切分 -> Embedding -> 写入向量库 -> 保存 metadata
```

在线链路：

```text
用户提问 -> Query Embedding -> 权限过滤 -> 召回 -> Rerank -> 拼 Prompt -> 大模型生成 -> 返回引用
```

### RAG 为什么不能完全消除幻觉？

因为检索可能召回错误资料，模型也可能忽略资料或过度推断。RAG 只是提供事实依据，仍然需要 Prompt 约束、引用、拒答、答案校验和评测。

### 如何优化 RAG 准确率？

- 改善文档解析质量。
- 优化 chunk 切分。
- 使用混合检索。
- 做 Query Rewrite。
- 召回后 Rerank。
- 优化 Prompt。
- 建立 Golden Dataset 评测。

## 2. Embedding

### Embedding 是什么？

Embedding 是把文本等对象映射成向量，使语义相近的内容在向量空间中距离更近。它是语义检索、推荐、聚类和 RAG 的基础。

### 向量检索和关键词检索区别？

关键词检索基于词面匹配，适合精确词、编号、错误码；向量检索基于语义相似，适合同义表达和模糊问法。生产中常组合使用。

### 为什么需要 metadata filter？

metadata 负责业务过滤，比如租户、知识库、权限、文档类型、时间范围。没有 metadata filter，向量检索可能召回用户无权限的数据。

## 3. Prompt

### Prompt 怎么设计？

一个完整 Prompt 包括角色、任务、上下文、约束、输出格式和示例。生产环境中还要做模板化、版本管理和回归评测。

### 如何让模型稳定输出 JSON？

- 使用低 temperature。
- 明确 JSON Schema。
- 使用结构化输出或 Tool Calling。
- 禁止输出额外文本。
- 后端做 JSON parse 和 schema 校验。
- 失败时重试或修复。

### Prompt 注入是什么？

用户通过恶意输入让模型忽略原系统指令、泄露信息或越权调用工具。防护不能只靠 Prompt，要把权限、工具校验和敏感信息过滤放在后端。

## 4. Tool Calling

### Tool Calling 是什么？

模型根据用户意图选择后端工具并生成参数，后端负责校验参数、鉴权和执行工具，最后模型基于工具结果生成回答。

### 为什么不能让模型直接执行 SQL？

模型可能生成错误 SQL、慢 SQL、越权查询甚至危险操作。应该通过后端封装工具访问数据库，工具内部做权限、参数、限流和审计。

### Tool Calling 如何保证安全？

- 工具白名单。
- JSON Schema 参数校验。
- 后端鉴权。
- 业务规则校验。
- 高风险操作人工确认。
- 工具调用日志。

## 5. Agent

### Agent 和 Workflow 区别？

Workflow 是预定义流程，可控、可测试，适合生产业务。Agent 由模型动态规划和选工具，灵活但不稳定，适合探索性任务。生产中通常用 Workflow 包住关键路径，局部引入 Agent。

### Agent 为什么难落地？

因为它的步骤、成本、延迟和工具调用结果都更不确定，调试和复现困难，还容易出现权限和安全风险。

## 6. 模型网关

### 为什么需要模型网关？

模型网关统一封装模型调用，处理路由、超时、重试、限流、降级、日志、成本统计和多供应商适配，避免业务代码到处直接调用模型。

### 模型调用失败怎么办？

- 超时控制。
- 可重试错误做指数退避。
- 熔断异常模型。
- 切备用模型。
- 返回缓存答案或兜底提示。
- 记录日志和告警。

### 如何降低模型成本？

- 模型分级。
- 缓存高频问题。
- Prompt 压缩。
- 控制 TopK。
- 控制 max_tokens。
- 批量 Embedding。
- 监控 token 成本。

## 7. 流式输出

### SSE 和 WebSocket 怎么选？

SSE 是服务端到客户端单向推送，基于 HTTP，适合大模型文本流式输出。WebSocket 是全双工，适合实时双向交互、语音对话和协作场景。

### 流式输出是否减少总耗时？

不一定。它主要降低首 token 体感等待时间，让用户更早看到内容，但完整答案生成时间可能差不多。

## 8. 评测

### 如何评估 RAG？

检索看 Recall@K、MRR、NDCG；生成看答案正确率、忠实性、引用准确率、幻觉率；工程看 P95 延迟、错误率和 token 成本。

### 为什么需要 Golden Dataset？

固定评测集可以让不同 Prompt、模型、检索策略之间可比较，也能防止改动带来隐藏回归。

## 9. 安全

### 如何防止用户检索别人的文档？

文档和 chunk 入库时保存 tenant_id、owner_id、role、acl 等 metadata。检索时根据当前用户权限构造 filter，只在有权限范围内召回，返回引用前再次校验。

### 模型输出为什么不可信？

模型可能幻觉、格式错误、泄露敏感信息或误解工具结果。后端必须做结构校验、业务校验、权限校验和敏感信息过滤。
