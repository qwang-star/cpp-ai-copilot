# 两个月学习与项目推进计划

## 总目标

两个月后，你要拥有一个能放进简历、能本地演示、能支撑面试追问的主项目：

```text
C++ 企业级 AI Copilot：知识库问答 + 业务流程助手
```

最终你要能演示：

```text
1. 上传一份企业制度文档。
2. 后台异步解析、切分、Embedding、写入向量库。
3. 用户基于文档提问。
4. 系统返回带引用的流式答案。
5. 用户追问“我的报销单审批到哪了？”
6. 系统通过 Tool Calling 查询后端业务表并回答。
7. 后台能看到模型调用日志、token 成本和 bad case。
```

---

## 学习节奏

建议每天分三块：

```text
知识学习  40%
项目实现  40%
复盘表达  20%
```

不要只看资料，也不要只写代码。每天都要问自己：

```text
今天学的这个知识点，在我的项目里用在哪里？
面试官追问时，我能不能用项目例子讲出来？
```

## 第 3 周：MQ 异步文档入库链路

### 本周目标

让上传文档后，后台能异步处理。

### 要学的知识

```text

重试

幂等
线程池
任务状态机
```

### 项目任务

```text
1. Docker Compose 加 RabbitMQ。
2. 上传文档后投递 document_parse 消息。
3. 实现 C++ Worker 消费消息。
4. Worker 读取文档内容。
5. 做简单文本解析，第一版可以先支持 .txt / .md。
6. 实现 chunk 切分。
7. 把 chunk 元数据写入 MySQL。
8. 处理完成后 documents.status 改为 parsed。
9. 消费失败时重试。
10. 用 document_id 做幂等，避免重复插入 chunk。
```

### 第一版降低难度

PDF 解析可以先不硬啃。

建议顺序：

```text
先支持 Markdown / txt
再支持 PDF
最后支持 OCR
```

这样你先把链路跑通，后面再增强解析能力。

### 本周可演示成果

```text
上传一个 markdown 文档。
后台 worker 自动切 chunk。
MySQL 中能看到 document_chunks。
文档状态从 uploaded -> parsing -> parsed。
```

### 面试沉淀

## 简历项目描述草稿

```text
C++ 企业级 AI Copilot：基于 RAG 与 Tool Calling 的企业知识库问答系统

项目描述：使用 C++20 构建企业级 AI 助手系统，支持企业文档上传、异步解析、Embedding 向量化、知识库问答、SSE 流式输出和业务流程查询。系统通过 MySQL 管理文档、会话和日志，通过 Redis 做缓存和限流，通过 MQ 异步处理文档入库任务，通过向量数据库实现语义检索，并封装模型网关统一处理大模型调用。

技术亮点：
1. 设计文档入库链路，使用 MQ 异步完成解析、切分、Embedding 和向量索引，提升上传接口响应速度。
2. 实现 RAG 在线问答链路，支持 metadata 权限过滤、混合检索、Rerank、Prompt 组装和引用来源返回。
3. 基于 SSE 实现大模型流式输出，降低用户首屏等待时间。
4. 通过 Tool Calling 查询报销单等业务流程，并在后端完成参数校验、鉴权和审计。
5. 封装模型网关，支持超时、重试、降级、限流、token 统计和调用日志。
6. 构建 Golden Dataset 评测集，对比不同检索策略的 Recall@K、引用准确率和答案正确性。
```
