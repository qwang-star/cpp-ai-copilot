# 项目架构初稿

## 1. 总体架构

```text
前端页面
  |
  | HTTP / SSE
  v
C++ API Server
  |
  |-- Auth Module
  |-- Document Module
  |-- Chat Module
  |-- RAG Module
  |-- Tool Module
  |-- Model Gateway
  |
  |-- MySQL
  |-- Redis
  |-- RabbitMQ
  |-- Qdrant
  |-- MinIO
  |
  v
LLM / Embedding API
```

## 2. 核心模块

### Auth Module

负责：

```text
用户登录
JWT 校验
用户身份解析
知识库权限判断
```

### Document Module

负责：

```text
创建知识库
上传文档
保存文档状态
投递解析任务
查询文档状态
```

### Document Worker

负责：

```text
消费 MQ 任务
解析文档
文本清洗
chunk 切分
调用 Embedding
写入向量库
更新文档状态
```

### Chat Module

负责：

```text
接收用户问题
保存用户消息
调用 RAG Module
调用 Model Gateway
SSE 流式返回
保存助手消息
```

### RAG Module

负责：

```text
query embedding
metadata 权限过滤
向量检索
关键词检索
合并去重
Rerank
Prompt 组装
引用来源生成
```

### Tool Module

负责：

```text
定义工具 Schema
校验工具参数
工具鉴权
查询业务表
记录工具调用日志
```

### Model Gateway

负责：

```text
统一模型调用
模型路由
超时
重试
降级
流式转发
token 统计
调用日志
```

## 3. 文档入库链路

```text
用户上传文档
  -> API Server 接收文件
  -> 保存原文件
  -> MySQL 插入 documents
  -> MQ 投递 document_parse 消息
  -> Worker 消费消息
  -> 解析文本
  -> 切分 chunk
  -> 调用 Embedding
  -> Qdrant upsert 向量
  -> MySQL 更新文档状态 ready
```

## 4. 在线问答链路

```text
用户提问
  -> JWT 鉴权
  -> 保存用户消息
  -> query embedding
  -> metadata filter
  -> 向量检索
  -> 关键词检索
  -> 合并去重
  -> Rerank
  -> 拼接 Prompt
  -> Model Gateway 调用 LLM
  -> SSE 流式返回
  -> 保存助手消息和引用
```

## 5. Tool Calling 链路

```text
用户问报销单状态
  -> 模型判断需要调用工具
  -> 返回工具名和参数
  -> 后端校验参数
  -> 后端做权限判断
  -> 查询 reimbursement_orders
  -> 工具结果返回给模型
  -> 模型生成自然语言回答
  -> 保存工具调用日志
```

## 6. 数据存储分工

```text
MySQL
  用户、知识库、文档、chunk 元数据、会话、消息、日志、评测数据

Redis
  权限缓存、热点问题缓存、限流、分布式锁、短期状态

RabbitMQ
  文档解析、Embedding、索引构建异步任务

Qdrant
  chunk embedding 和 metadata

MinIO
  原始 PDF、Markdown、图片等文件
```
