## 一句话说清楚这个项目

一个 C++ Web 服务器骨架。浏览器发 HTTP 请求 → 服务器收 → 解析 → 路由分发 → 返回 JSON 响应。

目前只有一个接口：`GET /health`，返回 `{"status":"ok"}`。

## 同一数据在不同文件里的名字对照表

读代码时最大的困惑："同一个东西，怎么到另一个文件就换名字了？"

| 数据      | main.cpp                   | 传给构造函数的参数名 | server 内部成员名 | server 内部怎么用                             |
| ------- | -------------------------- | ---------- | ------------ | ---------------------------------------- |
| 配置对象    | `config`                   | `config`   | `config_`    | `config_.host`, `config_.port`           |
| 路由表     | `create_app_router()` 的返回值 | `router`   | `router_`    | `router_.route(request)`                 |
| 日志器     | `logger`                   | `logger`   | `logger_`    | `logger_.info(...)`, `logger_.warn(...)` |
| 收到的原始文本 | `buffer`（char数组）           | -          | -            | 转成 string 传给 `parse_http_request(raw)`   |
| 请求对象    | -                          | -          | -            | `request.method`, `request.path`         |
| 响应对象    | -                          | -          | -            | `response.serialize()`                   |


**命名规律：传参名 = 成员名去掉下划线后缀 `_`。** 这是项目的命名约定。

## 按"一次请求的完整生命周期"追踪

不想按文件读？那就假装浏览器发了一个请求，顺着数据流追。

### 追踪 1：启动阶段（执行一次，程序开始时）

起点：main.cpp 第 10-18 行
  │
  ├→ AppConfig::load("config/app.env")
  │     └→ [[02_config配置模块逐行讲解]] 第 9 节
  │
  ├→ parse_log_level(config.log_level)
  │     └→ [[07_logger日志模块逐行讲解]] 第 8 节
  │
  ├→ create_app_router()
  │     └→ [[06_application模块逐行讲解]] 第 4 节
  │
  ├→ SimpleHttpServer(config, router, logger)
  │     └→ [[05_simple_server模块逐行讲解]] 第 5 节（构造函数）
  │
  └→ server.run()
        └→ [[05_simple_server模块逐行讲解]] 第 6 节（主循环）

### 追踪 2：收到 GET /health 请求（每次请求都走一遍）

起点：simple_server.cpp while(true) 循环
  │
  ├→ accept() — 接电话
  │     └→ [[05_simple_server模块逐行讲解]] 6.2 节 ①
  │
  ├→ recv() — 听内容，存入 buffer
  │     └→ [[05_simple_server模块逐行讲解]] 6.2 节 ②
  │
  ├→ parse_http_request(raw) — 字符串 → HttpRequest
  │     └→ [[04_http模块逐行讲解]] 第 9 节（逐行拆解）
  │
  ├→ router_.route(request) — 查路由表，找 Handler
  │     └→ [[03_router路由模块逐行讲解]] 8.3 节
  │
  ├→ Handler（lambda）执行 — 业务逻辑
  │     └→ [[06_application模块逐行讲解]] 4.3 节
  │     └→ HttpResponse::json(200, ...) → [[04_http模块逐行讲解]] 第 7 节
  │
  ├→ response.serialize() — HttpResponse → HTTP 文本
  │     └→ [[04_http模块逐行讲解]] 第 8 节
  │
  ├→ send() — 回复浏览器
  │     └→ [[05_simple_server模块逐行讲解]] 6.2 节 ⑤
  │
  ├→ close_socket() — 挂断
  │     └→ [[05_simple_server模块逐行讲解]] 6.2 节 ⑥
  │
  └→ logger_.info() — 记日志
        └→ [[07_logger日志模块逐行讲解]] 7.3 节

### 追踪 3：收到不存在的请求（返回 404）

parse_http_request → router_.route(request)
  │
  └→ routes_.find("GET /nonexistent") → 没找到
        └→ [[03_router路由模块逐行讲解]] 8.3 节 第 2 步
        └→ 返回 HttpResponse::json(404, {"error":"route_not_found"})

### 追踪 4：Handler 内部出错（返回 500）

router_.route(request) → Handler 执行 → 抛出异常
  │
  └→ catch (const std::exception& error)
        └→ [[05_simple_server模块逐行讲解]] 6.2 节 ④
        └→ logger_.error(...) → [[07_logger日志模块逐行讲解]] 7.3 节
        └→ HttpResponse::json(500, {"error":"internal_server_error"})

## 推荐的三种读法

### 读法 A：按数据流追（适合"我想搞懂整个系统怎么跑"）

1. 打开 `main.cpp`，看启动流程
2. 假装发了一个 `GET /health`
3. 按上面的"追踪 2"，从 `while(true)` 开始，一步一步追
4. 每遇到一个函数调用，就跳到对应的讲解文档看它是怎么实现的
5. 追到 `send()` 结束，再回头追一次 `GET /nonexistent` 看 404

### 读法 B：按文件读（适合"我想吃透每个模块"）

推荐顺序：`02_config → 04_http → 03_router → 06_application → 07_logger → 05_simple_server`

为什么这个顺序？因为 `simple_server` 调用了前面所有模块，最后读它能验证前面的理解。

### 读法 C：按知识点读（适合"我只想学某个 C++ 语法"）

看上面的"按 C++ 知识点索引"表，找到对应的讲解章节，直接跳过去看实战代码。
