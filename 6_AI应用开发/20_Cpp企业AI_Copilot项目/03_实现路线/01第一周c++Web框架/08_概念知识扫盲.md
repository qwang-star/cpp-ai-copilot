## 1. `ostringstream` 不写文件 —— "stream" ≠ "文件"

**常见误解：** 看到 `stream` 就以为跟文件有关。

**真相：** C++ 里 "stream（流）" 是一个通用概念——把数据**往某处送**或**从某处取**。至于"某处"是什么，取决于具体类型。

### 输出流家族

```text
ostream（输出流基类 — 定义了 <<）
├── ofstream         →  往磁盘文件写
├── ostringstream    →  往内存字符串写
└── cout             →  往屏幕/控制台写
```

三者用法一模一样（都用 `<<`），但**目的地完全不同**：

```cpp
// ofstream — 写到磁盘文件
std::ofstream file("output.txt");
file << "hello";         // → 写入 output.txt 这个文件

// ostringstream — 写到内存里的字符串
std::ostringstream oss;
oss << "hello";          // → 存在内存里，不碰磁盘
std::string result = oss.str();  // → 倒出来："hello"

// cout — 写到屏幕
std::cout << "hello";    // → 打印到控制台
```

### 输入流家族

```text
istream（输入流基类 — 定义了 >> 和 getline）
├── ifstream         →  从磁盘文件读
├── istringstream    →  从内存字符串读
└── cin              →  从键盘读
```

### 名字拆解

```text
o  string  stream    →   往  字符串  里写  的流
│    │        │
输出 字符串   流

i  f  stream         →   从  文件  里读  的流
│   │     │
输入 文件  流
```

`ofstream` 是往 **f**ile（文件）写，`ostringstream` 是往 **string**（字符串）写。差一个字，目的地完全不同。

### 你项目里的实际使用

| 文件 | 代码 | 往哪读/写 |
|------|------|----------|
| `http.cpp` | `ostringstream output; output << ...; return output.str();` | 内存字符串（序列化 HTTP 响应） |
| `http.cpp` | `istringstream input(raw); input >> ... ; getline(input, line);` | 从内存字符串读（解析 HTTP 请求） |
| `config.cpp` | `ifstream input(path); getline(input, line);` | 从磁盘文件读（读配置文件） |
| `logger.cpp` | `cout << ...` / `cerr << ...` | 屏幕/控制台（打印日志） |
