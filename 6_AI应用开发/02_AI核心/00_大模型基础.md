# 03 大模型基础

## 1. 大模型是什么

大模型可以理解为一个通过大量文本和多模态数据训练出来的“概率生成系统”。

它不是数据库，也不是规则引擎。它擅长：

- 理解自然语言。
- 生成文本。
- 总结归纳。
- 改写翻译。
- 代码生成。
- 信息抽取。
- 根据上下文推理。

它不擅长：

- 保证事实永远正确。
- 记住实时业务数据。
- 自动遵守企业权限。
- 稳定输出完全符合程序解析的格式。
- 在没有工具的情况下执行真实操作。

## 2. Transformer

必须掌握：

- Token。
- Tokenizer。
- Embedding。
- Position Encoding。
- Self-Attention。
- Multi-Head Attention。
- Feed Forward Network。
- LayerNorm。
- Encoder。
- Decoder。

一句话解释：

```text
Transformer 的核心是 Attention，让模型在处理当前 token 时，动态关注上下文中最相关的 token。
```

为什么重要：

- 支持并行训练。
- 能捕捉长距离依赖。
- 适合大规模预训练。

面试表达：

```text
传统 RNN 逐步处理序列，长距离依赖容易衰减。
Transformer 用自注意力机制让每个 token 都能和其他 token 建立关系，因此更适合大规模语言建模。
```

## 3. GPT、BERT 与生成式模型

GPT 类：

- Decoder-only。
- 自回归生成。
- 根据前文预测下一个 token。
- 适合聊天、写作、代码生成。

BERT 类：

- Encoder-only。
- 双向理解。
- 适合分类、匹配、抽取、向量表示。

T5 / BART 类：

- Encoder-Decoder。
- 适合文本到文本任务。

AI 应用开发里，通常重点关注 GPT 类生成模型和 Embedding 模型。

## 4. Token 与上下文窗口

Token 是模型处理文本的基本单位。

影响：

- 输入越长，成本越高。
- 输入越长，延迟越高。
- 超过上下文窗口会被截断或报错。
- 输出 token 也计费。

常见工程问题：

- 历史对话不能无限塞。
- RAG 召回 chunk 不能无限塞。
- Prompt 模板不能无限复杂。
- 工具 schema 太多也会消耗上下文。

优化策略：

- 历史消息裁剪。
- 对早期对话做摘要。
- 控制 RAG TopK。
- Prompt 压缩。
- 工具按场景动态加载。
- 对长文档做分层摘要。

## 5. 模型参数

必须掌握：

- temperature：控制随机性，越高越发散。
- top_p：核采样，控制候选 token 概率范围。
- max_tokens：最大输出长度。
- stop：停止词。
- seed：有些平台支持，用于增强可复现性。
- response_format / structured output：约束输出结构。
- tool_choice：控制工具调用策略。

应用建议：

- 客服、知识库问答：低 temperature。
- 创意写作：高 temperature。
- JSON 抽取：低 temperature + 结构化输出。
- 高风险业务：强约束 + 校验 + 人工确认。

## 6. 预训练、微调、RAG 的区别

```text
预训练
  -> 让模型具备通用语言能力

指令微调
  -> 让模型学会听人类指令

领域微调
  -> 让模型适应特定风格、格式、任务

RAG
  -> 不改模型参数，把外部知识检索出来塞进上下文
```

秋招回答重点：

- 企业知识库问答优先考虑 RAG。
- 当问题是“知识更新”和“私有数据”时，RAG 更灵活。
- 当问题是“固定任务格式”和“输出风格”时，可以考虑微调。
- 微调不能可靠解决实时知识和权限问题。

## 7. 幻觉

幻觉是模型生成了看似合理但不真实的内容。

原因：

- 模型本质是概率生成。
- 缺少外部事实依据。
- Prompt 约束不足。
- 检索材料错误。
- 用户问题本身模糊。

缓解：

- RAG。
- 引用来源。
- 低 temperature。
- 要求不知道就拒答。
- 输出后校验。
- LLM-as-a-Judge。
- 人工审核。

注意：

```text
RAG 只能缓解幻觉，不能完全消除幻觉。
```

## 8. 模型调用的工程问题

必须掌握：

- 延迟：模型生成慢。
- 成本：按 token 或调用计费。
- 稳定性：第三方服务可能超时。
- 随机性：同样输入可能不同输出。
- 格式错误：JSON 可能不合法。
- 安全：可能泄露敏感内容或被 Prompt 注入。

后端解法：

- 模型网关统一封装。
- 超时与重试。
- 降级到备用模型。
- 流式输出。
- 结构化输出和 JSON 校验。
- 日志和成本统计。
- 灰度发布。
