# 后端 AI 应用开发秋招知识库

## 1. 使用方式

这套资料按“秋招可复习顺序”整理，不是按技术名词随便堆。

如果你是想查具体知识点，优先进入：

[10_知识库_A到F模块/README.md](10_知识库_A到F模块/README.md)

这个目录按 A-F 六大模块整理，每个知识点会尽量说明：

```text
是什么
为什么重要
核心机制
后端 AI 应用场景
常见坑
面试回答
```

推荐顺序：

```text
00_导航
  -> 01_后端基础
  -> 02_AI核心
  -> 03_应用编排
  -> 04_工程化
  -> 05_项目面试
  -> 99_参考资料
```

你可以把它当成一条后端 AI 应用开发的主线：

```text
先会做后端服务
  -> 再理解大模型能力
  -> 再掌握 RAG / Tool / Agent
  -> 再补工程化、安全、评测
  -> 最后包装成项目和面试答案
```

## 2. 文件夹说明

### 00_导航

作用：建立全局地图。

包含：

- 岗位定位。
- 学习路线。
- 能力模型。
- 面试地图。

看完要能回答：

- 后端 AI 应用开发到底做什么？
- 和算法岗、普通后端岗有什么区别？
- 两个月应该按什么顺序准备？

### 01_后端基础

作用：把已有后端八股映射到 AI 应用场景。

包含：

- 网络、OS、MySQL、Redis、MQ、分布式。
- Java / Python / Go 路线。
- Web 框架。
- API 设计。
- 同步、异步、流式接口。
- 幂等和分层架构。

看完要能回答：

- 为什么文档解析要异步？
- SSE 和 WebSocket 怎么选？
- AI 问答结果能不能缓存？
- 向量库为什么不能替代 MySQL？
- 如何设计聊天和文档上传接口？

### 02_AI核心

作用：掌握 AI 应用开发的基础概念。

包含：

- 大模型基础。
- Transformer。
- Token 和上下文窗口。
- Prompt。
- 结构化输出。
- Embedding。
- 向量数据库。
- 文件解析、OCR、多模态。

看完要能回答：

- 大模型为什么会幻觉？
- Prompt 怎么设计？
- 如何让模型稳定输出 JSON？
- Embedding 和关键词搜索有什么区别？
- PDF / 表格为什么会影响 RAG 效果？

### 03_应用编排

作用：掌握 AI 应用的核心业务链路。

包含：

- RAG 知识库问答。
- 文档入库链路。
- 在线检索生成链路。
- Tool Calling。
- Agent。
- Workflow。

看完要能回答：

- RAG 离线和在线链路是什么？
- chunk 太大或太小有什么问题？
- Rerank 有什么用？
- 为什么不能让模型直接执行 SQL？
- Agent 和 Workflow 怎么取舍？

### 04_工程化

作用：把 Demo 变成可上线系统。

包含：

- 模型网关。
- 流式输出。
- 超时重试。
- 熔断降级。
- 数据工程。
- 评测和 LLMOps。
- 安全、权限、合规。
- 部署、推理服务、云原生基础。

看完要能回答：

- 模型挂了怎么办？
- 如何降低 token 成本？
- 如何评估 RAG 优化是否有效？
- 如何防 Prompt 注入？
- 自部署模型和第三方 API 怎么选？

### 05_项目面试

作用：把知识组织成秋招可讲项目。

包含：

- 企业知识库问答系统项目包装。
- 两个月冲刺计划。
- AI 应用高频面试题库。

看完要能回答：

- 三分钟讲清楚你的项目。
- 讲出项目架构图。
- 讲出数据库表设计。
- 讲出三个真实难点。
- 扛住 RAG、权限、稳定性、评测追问。

## 3. 当前知识体系覆盖范围

已覆盖：

- 后端基础与 AI 场景映射。
- 编程语言、Web 框架、API 设计。
- 大模型基础。
- Prompt 与结构化输出。
- Embedding 与向量数据库。
- 文件解析、OCR、多模态。
- RAG。
- Tool Calling。
- Agent / Workflow。
- 模型网关、流式输出、稳定性。
- 数据工程、评测、LLMOps。
- 安全、合规、权限。
- 部署、推理服务、云原生基础。
- 项目包装和面试题库。
- 两个月复习计划。

暂时不作为重点：

- 大模型预训练源码。
- RLHF 复杂数学推导。
- CUDA / TensorRT 深度优化。
- 分布式训练。
- 从零实现向量数据库。

原因：

```text
这些更偏算法、训练、推理优化或基础设施岗位。
后端 AI 应用开发秋招应优先保证应用工程链路完整。
```

## 4. 一套最终面试表达

你最终要能自然说出：

```text
我准备的方向是后端 AI 应用开发，重点是把大模型能力工程化接入业务系统。
我熟悉后端基础，包括 MySQL、Redis、MQ、网络和分布式稳定性；AI 应用上重点掌握 RAG、Prompt、Embedding、Tool Calling、Agent / Workflow；工程化上关注模型网关、流式输出、权限安全、评测、成本和可观测性。
我的主项目是企业知识库问答系统，包含文档上传解析、异步向量化、混合检索、Rerank、大模型生成、引用来源、权限过滤、模型调用降级和评测闭环。
```
