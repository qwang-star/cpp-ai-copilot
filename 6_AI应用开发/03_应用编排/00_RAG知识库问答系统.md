# 06 RAG 知识库问答系统

## 1. RAG 是什么

RAG 是 Retrieval-Augmented Generation，检索增强生成。

核心思想：

```text
先从外部知识库检索相关资料，再让大模型基于这些资料生成答案。
```

它解决的问题：

- 模型知识过时。
- 企业私有知识不在模型参数里。
- 模型容易幻觉。
- 答案需要引用来源。

## 2. RAG 完整链路

### 离线链路

```text
文档上传
  -> 文件存储
  -> 文档解析
  -> 文本清洗
  -> 文档切分
  -> 生成 Embedding
  -> 写入向量数据库
  -> 保存 metadata
  -> 更新文档状态 ready
```

### 在线链路

```text
用户提问
  -> 鉴权
  -> Query Rewrite
  -> Query Embedding
  -> metadata 权限过滤
  -> 向量召回
  -> 关键词召回
  -> 合并去重
  -> Rerank
  -> 拼接上下文
  -> 调用大模型
  -> 流式返回
  -> 保存引用和日志
```

## 3. 文档解析

文档类型：

- PDF。
- Word。
- Markdown。
- HTML。
- TXT。
- Excel。
- 图片。
- 扫描件。

难点：

- PDF 版式复杂。
- 表格结构容易丢失。
- 页眉页脚会污染内容。
- 扫描件需要 OCR。
- 图片、图表、公式难解析。
- 文档标题层级需要保留。

解析结果建议转成：

```text
结构化 Markdown
```

因为 Markdown 能保留标题、列表、表格、代码块等结构。

## 4. 文本清洗

清洗内容：

- 去页眉页脚。
- 去重复水印。
- 去无意义空白。
- 修复换行。
- 保留标题层级。
- 表格转 Markdown。
- 代码块保留缩进。
- 记录页码和位置。

原则：

```text
不要为了干净，把有用结构也洗掉。
```

## 5. Chunk 切分

常见策略：

- 固定长度切分。
- 按段落切分。
- 按标题层级切分。
- 递归字符切分。
- 语义切分。
- 滑动窗口 overlap。

关键取舍：

```text
chunk 太大
  -> 召回不精准
  -> 占用上下文
  -> 成本高

chunk 太小
  -> 语义不完整
  -> 上下文断裂
  -> 需要更多 chunk 才能回答
```

建议：

- 制度、文档类：按标题和段落切。
- FAQ 类：一问一答一 chunk。
- 代码类：按函数、类、文件结构切。
- 表格类：尽量保留表头和行上下文。

## 6. 检索

### 向量检索

适合同义表达和语义相似。

### 关键词检索

适合：

- 专有名词。
- 错误码。
- 编号。
- 法律条款。
- 产品型号。

### 混合检索

```text
向量召回 TopK
+ 关键词召回 TopK
  -> 合并
  -> 去重
  -> Rerank
```

## 7. Rerank

Rerank 的作用：

- 初步召回追求“别漏”。
- 重排追求“排准”。
- 把最相关材料放到 Prompt 前面。

常见方式：

- Cross-Encoder Rerank 模型。
- LLM Rerank。
- 规则加权：标题、时间、权限、文档类型。

面试表达：

```text
召回阶段可以取较大的 TopK 保证召回率，之后用 Rerank 提高精度，再只把最相关的少量 chunk 放入上下文。
```

## 8. Query Rewrite

为什么需要：

- 用户问题可能太短。
- 多轮对话中有代词。
- 用户表达不规范。
- 需要扩展同义词。

例子：

```text
用户：那报销呢？
历史：用户刚才问“出差交通费标准”
改写：出差交通费如何报销，需要哪些材料？
```

## 9. 答案生成

Prompt 要求：

- 只基于资料回答。
- 不知道就拒答。
- 给引用来源。
- 不要编造。
- 答案结构清晰。

输出示例：

```text
答案：员工出差交通费需要在出差结束后 7 个工作日内提交报销申请。

依据：
[1] 《差旅报销制度》第 3.2 节
[2] 《费用审批流程》第 5 页
```

## 10. RAG 评测

检索指标：

- Recall@K。
- Precision@K。
- MRR。
- NDCG。

生成指标：

- 答案正确性。
- 忠实性。
- 引用准确性。
- 幻觉率。
- 拒答准确率。

工程指标：

- P95 延迟。
- Token 成本。
- 检索耗时。
- 模型耗时。
- 错误率。

## 11. 常见问题与解法

### 召回不到

原因：

- 文档没解析好。
- chunk 切坏了。
- embedding 模型不适合。
- query 太短。
- TopK 太小。

解法：

- 改进解析和切分。
- Query Rewrite。
- 混合检索。
- 提高 TopK。
- 加入同义词和业务词典。

### 召回很多但答案不准

原因：

- 召回噪声多。
- Rerank 缺失。
- Prompt 没有约束。
- 模型忽略资料。

解法：

- Rerank。
- 控制进入 Prompt 的 chunk 数量。
- 引用约束。
- 答案校验。

### 用户问总结整篇文档

普通 TopK RAG 不适合，因为它只召回局部片段。

解法：

- 文档级摘要。
- 分层摘要。
- Map-Reduce 总结。
- 按章节逐步总结。
