# 03 部署、推理服务与云原生基础

## 1. 这一章的定位

后端 AI 应用开发不一定要求你训练模型，但要知道模型服务如何部署、如何调用、如何扩容、如何监控。

面试中常见问题：

- 你调用的是第三方模型还是本地模型？
- 如果本地部署模型，怎么对外提供服务？
- 模型推理慢怎么办？
- GPU 资源如何利用？
- 服务如何灰度和回滚？

## 2. 第三方 API 和自部署模型

### 第三方 API

优点：

- 接入快。
- 模型能力强。
- 不用维护 GPU。
- 适合快速做业务。

缺点：

- 成本受供应商影响。
- 数据出域风险。
- 可控性有限。
- 服务稳定性依赖外部。

### 自部署模型

优点：

- 数据可控。
- 可做私有化部署。
- 可定制。
- 成本在高调用量时可能更低。

缺点：

- 需要 GPU。
- 运维复杂。
- 模型效果和更新要自己负责。
- 推理优化门槛高。

面试表达：

```text
早期业务可以用第三方 API 快速验证；如果涉及强隐私、私有化交付或调用量很大，可以考虑自部署模型，并通过模型网关屏蔽两种调用方式差异。
```

## 3. 推理服务

推理服务负责把模型封装成可调用的服务。

常见形式：

- HTTP API。
- gRPC。
- OpenAI-compatible API。
- 批量推理接口。
- 流式生成接口。

常见框架和工具：

- vLLM。
- TensorRT-LLM。
- TGI。
- Triton Inference Server。
- Ollama。
- llama.cpp。

秋招不必深挖源码，但要知道它们解决：

- 模型加载。
- 请求调度。
- KV Cache。
- 批处理。
- 流式输出。
- GPU 显存管理。
- 并发推理。

## 4. 推理性能指标

必须掌握：

- 首 token 延迟。
- 总生成延迟。
- tokens/s。
- 并发数。
- 吞吐量。
- GPU 利用率。
- 显存占用。
- P95 / P99 延迟。

为什么首 token 延迟重要：

```text
用户点发送后多久看到第一个字，决定体感是否卡顿。
```

为什么 tokens/s 重要：

```text
决定长答案生成速度。
```

## 5. 推理优化思路

了解即可：

- 量化。
- KV Cache。
- Continuous Batching。
- Speculative Decoding。
- 模型裁剪。
- 多 GPU 并行。
- Prompt Cache。
- Prefix Cache。

后端可落地优化：

- 流式输出。
- 模型分级。
- 控制 max_tokens。
- 控制上下文长度。
- 缓存高频结果。
- 异步任务。
- 批量 Embedding。
- 超时降级。

## 6. 容器化部署

必须掌握：

- Docker 镜像。
- Dockerfile。
- 环境变量。
- 配置文件。
- 日志输出。
- 健康检查。
- 端口暴露。

AI 应用注意：

- API Key 不能写死在镜像里。
- 模型文件可能很大。
- GPU 容器需要驱动和 runtime。
- 日志要脱敏。
- 文件上传目录要挂载对象存储或持久卷。

## 7. Kubernetes 基础

了解：

- Pod。
- Deployment。
- Service。
- Ingress。
- ConfigMap。
- Secret。
- HPA。
- Rolling Update。
- Liveness Probe。
- Readiness Probe。

AI 场景：

- Chat Service 可以水平扩容。
- Worker 可以按队列堆积扩容。
- 模型推理服务受 GPU 资源限制。
- 灰度发布 Prompt / 模型 / 服务版本。

## 8. 发布与回滚

AI 应用需要灰度的不只有代码：

- 模型版本。
- Prompt 版本。
- Embedding 模型。
- Rerank 模型。
- 检索策略。
- Chunk 策略。

上线流程：

```text
离线评测通过
  -> 小流量灰度
  -> 监控效果和成本
  -> 扩大流量
  -> 异常回滚
```

回滚注意：

- Prompt 可快速回滚。
- 模型可路由回滚。
- Embedding 模型变更可能需要重建索引，回滚更复杂。

## 9. 面试高频题

### 自部署模型和 API 调用怎么选？

从数据安全、成本、效果、运维复杂度、调用量、私有化要求几个角度回答。早期用 API 快速验证，强隐私或大规模调用考虑自部署。

### 为什么 Embedding 模型升级麻烦？

因为不同模型生成的向量空间不同，不能直接混用。通常要重新生成向量并重建索引。
